# Async JavaScript & Event Loop

Σε fullstack συνεντεύξεις, πολλές JavaScript ερωτήσεις είναι “τι θα εκτυπωθεί πρώτο;”. Για να απαντήσεις, πρέπει να καταλαβαίνεις event loop.

## Synchronous code

Ο synchronous κώδικας τρέχει πρώτος, γραμμή-γραμμή.

```js
console.log("start");
console.log("end");
```

## setTimeout

Το `setTimeout` στέλνει callback σε task queue. Ακόμα και με `0ms`, δεν τρέχει πριν τελειώσει ο τρέχων synchronous κώδικας.

```js
setTimeout(() => console.log("timeout"), 0);
console.log("sync");
```

Πρώτα εμφανίζεται `sync`.

## Promises και microtasks

Τα promise callbacks μπαίνουν σε microtask queue. Τα microtasks συνήθως εκτελούνται πριν τα normal timer tasks.

```js
setTimeout(() => console.log("timeout"), 0);
Promise.resolve().then(() => console.log("promise"));
console.log("sync");
```

Σειρά:

1. `sync`
2. `promise`
3. `timeout`

## async/await

Το `await` σταματά προσωρινά τη συνέχεια της async function και αφήνει τον υπόλοιπο κώδικα να συνεχίσει.

```js
async function load() {
  console.log("before");
  await Promise.resolve();
  console.log("after");
}
```

Το `after` συνεχίζει σε microtask.
