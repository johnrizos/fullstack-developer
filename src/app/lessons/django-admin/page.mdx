# The Django Admin

Το **admin** είναι το feature που έκανε διάσημο το Django: ένα πλήρες, αυτόματο διαχειριστικό panel για τα δεδομένα σου, χωρίς να γράψεις σχεδόν τίποτα. Είναι ιδανικό για content management και internal tools.

## Ενεργοποίηση & superuser

Το admin είναι ήδη στο `INSTALLED_APPS` και στα `urls.py` ενός νέου project:

```python
# config/urls.py
from django.contrib import admin
from django.urls import path

urlpatterns = [
    path("admin/", admin.site.urls),
]
```

```bash
python manage.py createsuperuser   # φτιάξε διαχειριστή
python manage.py runserver
# → http://localhost:8000/admin
```

## Register ένα model

```python
# blog/admin.py
from django.contrib import admin
from .models import Post

admin.site.register(Post)
```

Μόλις το κάνεις register, αποκτάς **αυτόματα** πλήρες CRUD: λίστα, αναζήτηση, δημιουργία, επεξεργασία, διαγραφή — με σωστά forms βασισμένα στο model. Μηδέν κώδικας UI.

## Customization με ModelAdmin

```python
@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    list_display = ("title", "author", "published", "created_at")  # στήλες λίστας
    list_filter = ("published", "created_at")                       # φίλτρα στο πλάι
    search_fields = ("title", "body")                               # search box
    list_editable = ("published",)                                  # inline edit
    ordering = ("-created_at",)
    prepopulated_fields = {"slug": ("title",)}                      # auto slug
```

Με λίγες γραμμές `ModelAdmin` φτιάχνεις ένα ισχυρό διαχειριστικό: στήλες, φίλτρα, αναζήτηση, ταξινόμηση, inline editing.

## Inlines — σχετιζόμενα δεδομένα μαζί

```python
class CommentInline(admin.TabularInline):
    model = Comment
    extra = 1

@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    inlines = [CommentInline]   # επεξεργασία comments ΜΕΣΑ στο post
```

Τα inlines σου επιτρέπουν να επεξεργάζεσαι σχετιζόμενες εγγραφές (π.χ. comments ενός post) στην ίδια σελίδα με το parent.

## Πότε (δεν) το χρησιμοποιείς

Το admin είναι **για διαχειριστές/staff**, όχι για τελικούς χρήστες:

- ✅ Internal tools, content management, γρήγορο data entry, debugging δεδομένων.
- ❌ Public-facing UI — εκεί φτιάχνεις δικά σου views/templates.

Η πρόσβαση ελέγχεται από permissions (`is_staff`, `is_superuser`) — δες το επόμενο μάθημα.

## Πώς απαντάς στη συνέντευξη

«What makes the Django admin special?»

> The admin is an auto-generated CRUD interface for your models. After `createsuperuser` and registering a model with `admin.site.register`, you get list, search, create, edit and delete with no UI code. I customise it with a `ModelAdmin`: `list_display` for columns, `list_filter` and `search_fields` for finding records, `list_editable` for inline edits, and inlines to manage related rows on the same page. It's perfect for internal tools and content management, gated by staff permissions — but for public-facing UI I build my own views and templates instead.
