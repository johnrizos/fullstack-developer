# JavaScript DOM, Forms & Storage

Τώρα ενώνουμε όσα έμαθες:

- Επιλέγουμε elements από το DOM.
- Ακούμε το submit μιας φόρμας.
- Σταματάμε το default reload με `event.preventDefault()`.
- Αποθηκεύουμε απλά δεδομένα στο `localStorage`.

Το `localStorage` κρατά δεδομένα στον browser, ακόμα και αν κάνεις refresh.

## DOM selection

Με `document.querySelector` βρίσκεις element από τη σελίδα.

```js
const form = document.querySelector("#note-form");
const input = document.querySelector("#note");
```

Το `#note-form` σημαίνει element με id `note-form`. Αν το selector είναι λάθος, το αποτέλεσμα μπορεί να είναι `null`, άρα σε real app χρειάζεται προσοχή.

## Form submit

Όταν κάνεις submit μια HTML φόρμα, ο browser από default μπορεί να κάνει page reload. Με `event.preventDefault()` κρατάς τον έλεγχο στο JavaScript.

```js
form.addEventListener("submit", function (event) {
  event.preventDefault();
});
```

## Reading και writing values

Τα inputs έχουν `.value`. Τα text elements συχνά ενημερώνονται με `.textContent`.

```js
const note = input.value;
savedNote.textContent = note;
```

Για user-provided text, το `textContent` είναι πιο ασφαλές από `innerHTML`, γιατί δεν εκτελεί HTML.

## localStorage

Το `localStorage` αποθηκεύει strings.

```js
localStorage.setItem("note", "Hello");
const note = localStorage.getItem("note");
localStorage.removeItem("note");
```

Μην βάζεις secrets, passwords ή sensitive tokens στο localStorage. Είναι αποθήκευση στον browser, όχι secure vault.

### Αποθήκευση objects: JSON.stringify / JSON.parse

Το localStorage δέχεται **μόνο strings**. Για arrays/objects, τα μετατρέπεις:

```js
const notes = [{ text: "Hello", createdAt: Date.now() }];

localStorage.setItem("notes", JSON.stringify(notes)); // object → string

const saved = JSON.parse(localStorage.getItem("notes") ?? "[]"); // string → object
```

Δύο λεπτομέρειες που γλιτώνουν bugs: το `getItem` επιστρέφει `null` αν δεν υπάρχει το key (γι' αυτό το `?? "[]"`), και το `JSON.parse` πετάει exception σε χαλασμένα δεδομένα — σε πραγματικό app το τυλίγεις σε try/catch.

## Event delegation: ένας listener για πολλά στοιχεία

Όταν έχεις λίστα με κουμπιά (π.χ. delete ανά note), μη βάζεις listener σε καθένα — βάλε **έναν στον γονέα** και δες ποιος πατήθηκε:

```js
list.addEventListener("click", (event) => {
  const button = event.target.closest("[data-delete-id]");
  if (!button) return;
  deleteNote(button.dataset.deleteId);
});
```

Δουλεύει επειδή τα events κάνουν **bubbling** προς τα πάνω — και λειτουργεί αυτόματα και για στοιχεία που θα προστεθούν αργότερα. Κλασική ερώτηση συνέντευξης για DOM.

### Capturing & bubbling: οι δύο φάσεις

Κάθε event ταξιδεύει σε **δύο φάσεις**. Πρώτα η **capture** φάση: κατεβαίνει από το `document` προς τα κάτω, μέχρι το στοιχείο-στόχο. Μετά η **bubble** φάση: ανεβαίνει πάλι πίσω στους γονείς. Από προεπιλογή οι listeners τρέχουν στη **bubble** φάση — γι' αυτό δουλεύει το delegation παραπάνω.

Αν θες να πιάσεις το event στο **κατέβασμα** (capture), δίνεις τρίτο όρισμα `true`:

```js
parent.addEventListener("click", handler, true);  // capture φάση — τρέχει ΠΡΙΝ τα παιδιά
parent.addEventListener("click", handler);          // bubble φάση (default) — ΜΕΤΑ τα παιδιά
```

Σχετικά εργαλεία μέσα στον handler:

- `event.stopPropagation()` — σταματά τη διάδοση, δεν συνεχίζει σε γονείς (χρήσιμο αλλά «σπάει» το delegation, χρησιμοποίησέ το με μέτρο).
- `event.target` — το στοιχείο που **πραγματικά** πατήθηκε· `event.currentTarget` — το στοιχείο στο οποίο **κρέμασες** τον listener.

## Mini checklist

- Το form έχει submit listener.
- Το submit δεν κάνει reload.
- Η τιμή διαβάζεται από το input.
- Η σελίδα ενημερώνεται αμέσως.
- Το data παραμένει μετά από refresh.
