# States, Variants & Dark Mode

Στο κανονικό CSS γράφεις `:hover` blocks και `@media (prefers-color-scheme)`. Στο Tailwind όλα αυτά γίνονται **prefixes** πάνω σε utilities. Αυτό το μάθημα καλύπτει state variants, τις σχέσεις `group`/`peer`, και το dark mode.

## State variants

Με prefix εφαρμόζεις utility υπό κατάσταση, χωρίς να γράψεις `:hover` block:

```html
<button class="bg-blue-600 hover:bg-blue-700 focus:ring-2 active:scale-95 disabled:opacity-50">
  Κουμπί
</button>
```

- `hover:` → στο hover
- `focus:` / `focus-visible:` → στο focus (κράτα το για accessibility!)
- `active:` → στο πάτημα
- `disabled:` → όταν `disabled`

Κάθε variant ισχύει **μόνο** όταν συμβαίνει η κατάσταση.

## Συνδυασμός variants

Τα variants **στοιβάζονται**: responsive + state μαζί.

```html
<!-- σκουραίνει στο hover ΜΟΝΟ από md κι άνω -->
<button class="bg-blue-600 md:hover:bg-blue-700">OK</button>
```

Η σειρά: `responsive:state:utility` (π.χ. `md:hover:bg-blue-700`).

## group & peer — αντιδράσεις σε «γείτονες»

Συχνά θες ένα παιδί να αλλάζει όταν κάνεις hover στον **γονιό**. Αυτό είναι το `group`:

```html
<div class="group rounded-lg p-4 hover:bg-gray-100">
  <h3 class="text-gray-900">Τίτλος</h3>
  <p class="text-gray-500 group-hover:text-gray-900">Σκουραίνει με hover στην κάρτα</p>
</div>
```

Βάζεις `group` στον γονιό, και `group-hover:` στο παιδί. Το `peer` κάνει το ίδιο για **αδέλφια** (π.χ. ένα label που αντιδρά στο focus ενός input):

```html
<input class="peer" />
<p class="hidden peer-invalid:block text-red-600">Άκυρο input</p>
```

## Dark mode

Το Tailwind δίνει το `dark:` variant. Στρατηγική **class** (η πιο συνηθισμένη): το dark ενεργοποιείται όταν υπάρχει class `dark` στο `<html>`:

```html
<div class="bg-white text-gray-900 dark:bg-gray-900 dark:text-gray-100">
  Προσαρμόζεται στο theme
</div>
```

```js
// Toggle: πρόσθεσε/αφαίρεσε την class "dark" στο root
document.documentElement.classList.toggle("dark");
```

Κάθε `dark:` utility ισχύει μόνο όταν είναι ενεργό το dark theme. Για να μη γράφεις `dark:` σε κάθε χρώμα, συνδύασέ το με **σημασιολογικά χρώματα** στο config (επόμενο μάθημα) — ώστε το ίδιο όνομα να σημαίνει «φόντο» σε κάθε theme.

## Πώς απαντάς στη συνέντευξη

> «How do you handle interactive states and dark mode in Tailwind?»

> States are variants: `hover:`, `focus:`, `active:`, `disabled:` prefix a utility so it applies only in that state, and they stack with responsive prefixes like `md:hover:`. For relationships I use `group` — put `group` on the parent and `group-hover:` on a child so the child reacts to hovering the whole parent — and `peer` for sibling state. Dark mode is the `dark:` variant: with the class strategy it activates when `<html>` has a `dark` class, which I toggle in JS, and I pair it with semantic colours in the theme config so I'm not writing `dark:` on every element.
