# Bootstrap Foundations

Το **Bootstrap** είναι το πιο γνωστό component-first CSS framework — τρέχει πίσω από τεράστιο μέρος του web, ειδικά admin panels, dashboards, εσωτερικά tools και prototypes. Αυτό το πρώτο μάθημα χτίζει το θεμέλιο: τη **φιλοσοφία** component-first, το setup (CSS + JS), και το πρότυπο χρήσης με semantic component classes.

## Component-first — η φιλοσοφία

Αντί για μικρά utilities που συναρμολογείς (Tailwind), το Bootstrap σου δίνει **έτοιμα, στιλιζαρισμένα components**. Δεν ορίζεις εσύ padding/χρώμα/radius — βάζεις την «επώνυμη» class και παίρνεις έτοιμο στυλ:

```html
<!-- Tailwind: px-4 py-2 bg-blue-600 text-white rounded ... -->
<!-- Bootstrap: λες "είναι ένα primary button" -->
<button class="btn btn-primary">Αποθήκευση</button>
```

Το `.btn` δίνει τη βάση (padding, border, cursor), το `.btn-primary` το χρώμα variant. Core trade-off: **ταχύτητα & συνέπεια** έναντι **ελευθερίας**. Παίρνεις γρήγορα ένα ολοκληρωμένο, συνεπές UI «out of the box», αλλά παίζεις με τα components του.

## Setup: CSS + JavaScript

Δύο τρόποι: **CDN** (γρήγορα, για prototype) ή **npm** (για build). Το Bootstrap έχει **δύο** κομμάτια:

```html
<!-- CSS: όλα τα styles των components + grid (υποχρεωτικό) -->
<link href="bootstrap.min.css" rel="stylesheet" />

<!-- ...το markup σου... -->

<!-- JS bundle: για interactive components (modal, dropdown). Προαιρετικό -->
<script src="bootstrap.bundle.min.js"></script>
```

Το **CSS** αρκεί για τα στατικά components (buttons, cards, alerts, grid). Το **JavaScript** χρειάζεται μόνο για τα interactive (modals, dropdowns — τα βλέπεις σε ξεχωριστό μάθημα).

## Το πρότυπο: variant classes

Πολλά components ακολουθούν το μοτίβο **base class + variant**:

```html
<button class="btn btn-primary">Primary</button>
<button class="btn btn-outline-secondary">Outline</button>
<button class="btn btn-danger btn-lg">Large danger</button>

<div class="alert alert-success">Επιτυχία!</div>
<div class="alert alert-warning">Προσοχή!</div>

<span class="badge bg-danger">3</span>
```

Τα contextual χρώματα (`primary`, `secondary`, `success`, `danger`, `warning`, `info`) επαναλαμβάνονται σε όλα τα components — μαθαίνεις το λεξιλόγιο μία φορά.

## Πότε λάμπει το Bootstrap

- **Admin panels, dashboards, εσωτερικά tools** — standard UI χωρίς custom design.
- **Prototypes** — από ιδέα σε λειτουργικό UI σε λεπτά.
- Ομάδες χωρίς dedicated designer που θέλουν συνεπές, προσβάσιμο αποτέλεσμα.

Λιγότερο ιδανικό όταν θες **μοναδικό, custom design** — εκεί ταιριάζει το Tailwind.

## Πώς απαντάς στη συνέντευξη

> «What is Bootstrap and when would you use it?»

> Bootstrap is a component-first framework: instead of composing utilities, I use prebuilt, styled, accessible components like `btn btn-primary`, `card`, and `alert`, plus a responsive grid. It ships CSS for the static components and an optional JavaScript bundle for interactive ones like modals. Components follow a base-plus-variant pattern — `btn` plus `btn-primary` — and contextual colours repeat across components, so I learn the vocabulary once. It's the fastest way to ship a consistent, functional UI without designing from scratch, so I reach for it on admin panels, dashboards, internal tools and prototypes; for a unique custom design I'd pick Tailwind.
