# Logs, Monitoring & Observability

Όταν κάτι σπάσει στην παραγωγή, δεν έχεις debugger πάνω στον χρήστη. Χρειάζεσαι observability.

## Logs

Logs πρέπει να απαντούν:

- τι συνέβη,
- πότε,
- για ποιο request/user/session,
- με ποιο error.

Καλύτερα structured logs:

```json
{
  "level": "error",
  "requestId": "abc123",
  "message": "Payment failed"
}
```

## Error tracking

Εργαλεία όπως Sentry βοηθούν να βλέπεις stack traces, affected users και frequency.

## Metrics

Metrics δείχνουν patterns:

- latency,
- error rate,
- request count,
- memory/CPU.
