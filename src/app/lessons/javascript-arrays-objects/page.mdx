# JavaScript Arrays & Objects

Σχεδόν κάθε πραγματική εφαρμογή είναι «λίστες από πράγματα»: προϊόντα, χρήστες, tasks, μηνύματα. Τα **arrays** κρατούν τις λίστες, τα **objects** περιγράφουν το κάθε «πράγμα» — και τα array methods είναι το εργαλείο που μετατρέπει δεδομένα σε UI. Αυτό το pattern θα το χρησιμοποιείς καθημερινά, από vanilla JavaScript μέχρι React.

## Objects: περιγράφουν ένα πράγμα

```js
const task = {
  id: 1,
  title: "Μάθε JavaScript",
  done: false,
  tags: ["basics", "logic"], // τα values μπορούν να είναι ΚΑΙ arrays/objects
};

task.title;          // "Μάθε JavaScript" — dot notation
task["title"];       // ίδιο — bracket notation (χρήσιμο για δυναμικά keys)
task.done = true;    // αλλαγή τιμής
task.priority = 2;   // προσθήκη νέου property
```

Δύο εργαλεία που κάνουν τον κώδικα πολύ πιο καθαρό:

```js
// Destructuring: "ξεπακετάρισμα" σε μεταβλητές
const { title, done } = task;

// Spread: αντίγραφο με αλλαγές — ΧΩΡΙΣ να πειράξεις το αρχικό
const updatedTask = { ...task, done: true };
```

Το spread pattern (`{ ...task, done: true }`) είναι η βάση των immutable updates — θα το ξαναδείς παντού στο React.

## Arrays: κρατούν τη λίστα

```js
const names = ["Ada", "Grace", "Linus"];

names.length;     // 3
names[0];         // "Ada" — μηδενική αρίθμηση
names.push("X");  // προσθήκη στο τέλος (αλλάζει το array)
names.includes("Ada"); // true

// Destructuring και spread δουλεύουν κι εδώ:
const [first, second] = names;
const copy = [...names];
const merged = [...names, ...otherNames];
```

### Rest: το αντίστροφο του spread

Το ίδιο σύμβολο `...` έχει **δεύτερη χρήση**. Όταν είναι στη μεριά της λήψης (αριστερά σε destructuring ή ως παράμετρος συνάρτησης), **μαζεύει** ό,τι περισσεύει σε ένα array — γι' αυτό λέγεται *rest*:

```js
// rest σε array destructuring — "πρώτο και όλα τα υπόλοιπα"
const [winner, ...others] = ["Ada", "Grace", "Linus"];
winner; // "Ada"
others; // ["Grace", "Linus"]

// rest σε object destructuring — "βγάλε το password, κράτα τα υπόλοιπα"
const { password, ...safeUser } = user;

// rest parameters — δέξου άγνωστο πλήθος ορισμάτων ως array
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
sum(1, 2, 3); // 6
```

Ο κανόνας για να μην μπερδεύεσαι: **spread = «άπλωσε/σκόρπισε»** (το `...` σε τιμή που δίνεις), **rest = «μάζεψε τα υπόλοιπα»** (το `...` εκεί που λαμβάνεις). Το rest πάει **πάντα τελευταίο**.

Η πιο χρήσιμη μορφή στα πραγματικά apps: **array από objects** —

```js
const tasks = [
  { id: 1, title: "Μάθε HTML", done: true },
  { id: 2, title: "Μάθε CSS", done: true },
  { id: 3, title: "Μάθε JavaScript", done: false },
];
```

## Τα array methods που θα γράφεις κάθε μέρα

Όλα παίρνουν μια function και **επιστρέφουν αποτέλεσμα χωρίς να αλλάζουν το αρχικό array**:

```js
// map: μετασχηματίζει ΚΑΘΕ item → νέο array ΙΔΙΟΥ μήκους
const titles = tasks.map((task) => task.title);
// ["Μάθε HTML", "Μάθε CSS", "Μάθε JavaScript"]

// filter: κρατά όσα περνούν το τεστ → νέο (ίσως μικρότερο) array
const active = tasks.filter((task) => !task.done);

// find: το ΠΡΩΤΟ item που ταιριάζει (ή undefined)
const css = tasks.find((task) => task.id === 2);

// some / every: υπάρχει έστω ένα; / ισχύει για όλα;
tasks.some((task) => task.done);   // true
tasks.every((task) => task.done);  // false

// reduce: "συμπυκνώνει" το array σε ΜΙΑ τιμή
const doneCount = tasks.reduce((count, task) => (task.done ? count + 1 : count), 0);
```

Πώς διαλέγεις method; Ρώτα τι θες πίσω:

- Ίδιο πλήθος, αλλαγμένα items → `map`
- Λιγότερα items → `filter`
- Ένα item → `find`
- true/false → `some` / `every`
- Ένα «σύνολο» (άθροισμα, object, οτιδήποτε) → `reduce`

Και φυσικά συνδυάζονται σε αλυσίδες:

```js
const activeTitles = tasks
  .filter((task) => !task.done)
  .map((task) => task.title);
```

## Map & Set: όταν το object/array δεν φτάνει

Δύο δομές που λύνουν συγκεκριμένα προβλήματα — και ρωτιούνται σε συνεντεύξεις.

**`Set`** = συλλογή με **μοναδικές** τιμές. Ιδανικό για dedup και για «υπάρχει;» έλεγχο:

```js
const ids = new Set([1, 2, 2, 3]); // {1, 2, 3} — τα διπλά φεύγουν μόνα τους
ids.add(4);
ids.has(2);   // true — γρήγορος έλεγχος ύπαρξης
ids.size;     // 4

const unique = [...new Set([1, 1, 2, 3])]; // [1, 2, 3] — το πιο καθαρό dedup
```

**`Map`** = συλλογή από ζεύγη key→value, σαν object αλλά **καλύτερο για δυναμικά keys**:

```js
const scores = new Map();
scores.set("ada", 95);
scores.get("ada");   // 95
scores.has("ada");   // true
scores.size;         // 1
for (const [name, score] of scores) { /* ... */ }
```

Πότε `Map` αντί για `{}`;

- **Keys οποιουδήποτε τύπου** — όχι μόνο strings (ακόμα και objects ως keys).
- **Πραγματικό `size`** και εύκολο iteration με σειρά εισαγωγής.
- **Καθαρό κλειδί από user input** — ένα `{}` έχει κληρονομημένα keys (`toString`, `constructor`) που μπερδεύουν τους ελέγχους· το `Map` όχι.

Για σταθερά, γνωστά πεδία ενός «πράγματος» → object. Για δυναμικό lookup table που μεγαλώνει/συρρικνώνεται → `Map`.

## Προσοχή: references, όχι αντίγραφα

Τα objects και τα arrays **δεν αντιγράφονται** με την ανάθεση — η μεταβλητή δείχνει στο ίδιο πράγμα:

```js
const a = { count: 1 };
const b = a;        // ΟΧΙ αντίγραφο — ίδιο object!
b.count = 99;
console.log(a.count); // 99 😱

const realCopy = { ...a }; // αυτό είναι αντίγραφο (πρώτου επιπέδου)
```

Το ίδιο ισχύει και στις συγκρίσεις: `[1,2] === [1,2]` είναι `false` — συγκρίνονται οι references, όχι τα περιεχόμενα. Κράτα το στο μυαλό σου· θα το εξερευνήσουμε σε βάθος στα interview drills.

### Shallow vs deep copy

Το `{...a}` και το `[...arr]` είναι **shallow** (ρηχό) αντίγραφο: αντιγράφουν το πρώτο επίπεδο, αλλά τα **nested** objects/arrays παραμένουν κοινές references:

```js
const original = { user: { name: "Ada" } };
const shallow = { ...original };
shallow.user.name = "Grace";
console.log(original.user.name); // "Grace" 😱 — το nested object είναι κοινό!
```

Για **deep** (βαθύ) αντίγραφο, εντελώς ανεξάρτητο, χρησιμοποίησε `structuredClone`:

```js
const deep = structuredClone(original);
deep.user.name = "Linus";
console.log(original.user.name); // αμετάβλητο — πραγματικά ξεχωριστό
```

### `Object.freeze` — «κλείδωσε» ένα object

Αν θες να αποτρέψεις αλλαγές (π.χ. σε config/constants), το `Object.freeze` κάνει το object read-only (στο πρώτο επίπεδο):

```js
const config = Object.freeze({ apiUrl: "/api", retries: 3 });
config.retries = 5;        // αγνοείται σιωπηλά (ή πετάει error σε strict mode)
console.log(config.retries); // 3
```

> Σημείωση: το `const` εμποδίζει το **re-assignment** της μεταβλητής, ΟΧΙ τη μεταβολή του object. Το `Object.freeze` εμποδίζει τη μεταβολή των properties. Είναι δύο διαφορετικά πράγματα.

## Rendering: από δεδομένα σε UI

Το θεμελιώδες frontend pattern, που ισχύει από vanilla JS μέχρι React:

1. Έχεις **array δεδομένων** (το source of truth).
2. Το **μετασχηματίζεις** σε HTML/components με `map`.
3. Όταν αλλάξουν τα δεδομένα, **ξανακάνεις render** — δεν «μπαλώνεις» το UI με το χέρι.

```js
function renderTasks() {
  list.innerHTML = tasks
    .map((task) => `<li>${task.done ? "✅" : "⬜"} ${task.title}</li>`)
    .join(""); // το map δίνει array από strings — το join τα ενώνει
}
```

> Σημείωση ασφαλείας: το `innerHTML` είναι μια χαρά για δικά σου δεδομένα, αλλά **ποτέ** για κείμενο που έγραψε χρήστης — εκεί ελλοχεύει το XSS. Θα το δεις αναλυτικά στο Web Security μάθημα.
