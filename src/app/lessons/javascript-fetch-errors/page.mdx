export const fetchJs = `const status = document.querySelector("#status");
const result = document.querySelector("#result");

async function loadUser() {
  status.textContent = "Loading...";
  result.textContent = "";

  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/users/1");

    if (!response.ok) {
      throw new Error("Request failed: " + response.status);
    }

    const user = await response.json();
    status.textContent = "Loaded successfully.";
    result.textContent = JSON.stringify(user, null, 2);
  } catch (error) {
    status.textContent = "Something went wrong.";
    result.textContent = error.message;
  }
}

document.querySelector("#load").addEventListener("click", loadUser);`;

# Fetch, Errors & API Handling

Fullstack developers δουλεύουν συνέχεια με APIs. Σε συνεντεύξεις δεν αρκεί να πεις “χρησιμοποιώ fetch”. Πρέπει να ξέρεις τι γίνεται όταν:

- το request αργεί,
- το API επιστρέφει `404` ή `500`,
- το JSON parsing αποτυγχάνει,
- πρέπει να δείξεις loading και error state.

## fetch

Το `fetch` στέλνει HTTP request και επιστρέφει Promise.

```js
const response = await fetch("/api/users");
```

## response.ok

Σημαντικό: το `fetch` δεν κάνει throw αυτόματα για HTTP errors όπως `404` ή `500`. Πρέπει να ελέγξεις το `response.ok`.

```js
if (!response.ok) {
  throw new Error("Request failed");
}
```

## try/catch

Το `try/catch` πιάνει network errors, thrown errors και async failures μέσα σε `await`.

```js
try {
  const data = await loadData();
} catch (error) {
  console.error(error);
}
```

## Loading και error states

Σε UI, σχεδόν πάντα χρειάζεσαι:

- loading state πριν έρθει απάντηση,
- success state όταν έρθουν data,
- error state όταν κάτι πάει λάθος.
