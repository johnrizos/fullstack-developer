# Node.js, Express & NestJS

Backend developer πρέπει να ξέρει πώς στήνεται server, πώς ορίζονται routes και πώς περνάει ένα request μέσα από middleware μέχρι να επιστραφεί response.

## Node.js

Node.js επιτρέπει να τρέχεις JavaScript στον server. Εκεί χειρίζεσαι:

- HTTP requests,
- business logic,
- database calls,
- authentication,
- background tasks.

Το σημαντικό mental model: ο browser δεν έχει άμεση πρόσβαση στη database. Ο browser κάνει request στον server, ο server ελέγχει auth/validation, μιλάει με database ή τρίτα APIs και επιστρέφει response.

Τυπική ροή:

1. Client στέλνει `GET /api/tasks`.
2. Middleware ελέγχει logging/auth.
3. Route handler διαβάζει query params.
4. Service εφαρμόζει business logic.
5. Database layer φέρνει δεδομένα.
6. Server επιστρέφει JSON.

## Express

Express είναι minimal framework.

```ts
app.get("/tasks", async (req, res) => {
  res.json({ data: [] });
});
```

Πιο ρεαλιστικό παράδειγμα:

```ts
app.post("/tasks", async (req, res) => {
  const title = req.body.title;

  if (typeof title !== "string" || title.trim().length < 2) {
    return res.status(400).json({ error: "Title is required" });
  }

  const task = await taskService.createTask({ title });
  return res.status(201).json({ data: task });
});
```

Το route handler δεν πρέπει να γεμίζει με όλη τη λογική. Συνήθως κρατάς validation/request parsing εκεί και στέλνεις τη δουλειά σε service.

## Middleware

Middleware τρέχει πριν από route handler.

Χρήσεις:

- logging,
- auth checks,
- parsing JSON,
- validation,
- error handling.

Παράδειγμα request logger:

```ts
app.use((req, _res, next) => {
  console.info({ method: req.method, path: req.path });
  next();
});
```

Παράδειγμα error handler:

```ts
app.use((error, _req, res, _next) => {
  console.error(error);
  res.status(500).json({ error: "Internal server error" });
});
```

## NestJS

NestJS είναι πιο structured Node framework. Χρησιμοποιεί modules, controllers, services και dependency injection. Σε μεγαλύτερες ομάδες εμφανίζεται συχνά επειδή οργανώνει καλύτερα μεγάλα backends.

Στο NestJS η ίδια ιδέα χωρίζεται πιο καθαρά:

- **Controller**: δέχεται HTTP request.
- **Service**: περιέχει business logic.
- **Module**: οργανώνει related pieces.
- **Guard**: ελέγχει auth/permissions.
- **Pipe**: κάνει validation/transformation.

```ts
@Controller("tasks")
export class TasksController {
  constructor(private readonly tasksService: TasksService) {}

  @Get()
  findAll() {
    return this.tasksService.findAll();
  }
}
```

## Mini project: small tasks API

Σχεδίασε backend API για tasks:

1. `GET /tasks` επιστρέφει λίστα.
2. `POST /tasks` κάνει validation και δημιουργεί task.
3. `PATCH /tasks/:id` αλλάζει `completed`.
4. Middleware γράφει method/path σε log.
5. Error handler επιστρέφει predictable JSON.

## Backend checklist

- Μην εμπιστεύεσαι input από client.
- Κράτα secrets server-side.
- Χώρισε route handlers από business logic.
- Χρησιμοποίησε σωστά status codes.
- Πρόσθεσε centralized error handling.
