# Generics, keyof & Indexed Access

Τα generics είναι «παράμετροι τύπων»: γράφεις κώδικα που δουλεύει με **οποιονδήποτε** τύπο, διατηρώντας την τυποασφάλεια. Μαζί με τα `keyof`, `typeof` (type operator) και τα indexed access types (`T[K]`), αποτελούν το «βαρύ οπλοστάσιο» που στηρίζει τα utility types και τις βιβλιοθήκες — και είναι από τα πιο διακριτικά interview topics.

## Generics — επαναχρησιμοποιήσιμα types

Χωρίς generics, θα έγραφες το ίδιο type ξανά και ξανά (ή θα κατέφευγες σε `any`, χάνοντας την ασφάλεια):

```ts
interface ApiResponse<T> {
  data: T;
  error?: string;
}

type TasksResponse = ApiResponse<Task[]>;
type UserResponse = ApiResponse<User>;

// Σε functions — με inference, δεν δηλώνεις το T:
function first<T>(items: T[]): T | undefined {
  return items[0];
}
const task = first(tasks);  // T = Task, αυτόματα
```

Το `<T>` είναι placeholder που «γεμίζει» τη στιγμή της χρήσης. Σημαντικό: το generic **διατηρεί** τη σχέση input→output — η `first` επιστρέφει ακριβώς τον τύπο των στοιχείων, όχι `any`.

## Constraints — `extends`

Με constraints περιορίζεις τι δέχεται το generic, ώστε να μπορείς να το χρησιμοποιήσεις με ασφάλεια:

```ts
function getById<T extends { id: string }>(items: T[], id: string): T | undefined {
  return items.find((item) => item.id === id); // ✅ ξέρει ότι κάθε T έχει id
}

getById(tasks, "1");      // ✅ τα Task έχουν id
getById([1, 2, 3], "1");  // ❌ τα numbers δεν έχουν id
```

Το `T extends { id: string }` σημαίνει «οποιοσδήποτε τύπος που **έχει τουλάχιστον** `id: string`».

## `keyof` — τα κλειδιά ενός τύπου ως union

Το `keyof` παίρνει ένα object type και δίνει union από τα **κλειδιά** του:

```ts
interface Task { id: string; title: string; done: boolean }
type TaskKey = keyof Task; // "id" | "title" | "done"
```

## Indexed access types — `T[K]`

Παίρνεις τον τύπο μιας **property** μέσω του κλειδιού της:

```ts
type TitleType = Task["title"];        // string
type Value = Task["id" | "done"];      // string | boolean
```

## Το κλασικό combo: type-safe property getter

`keyof` + indexed access + generics μαζί δίνουν μια απόλυτα typed `getProp`:

```ts
function getProp<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const task: Task = { id: "1", title: "Μάθε TS", done: false };
const title = getProp(task, "title");  // τύπος: string
const done = getProp(task, "done");    // τύπος: boolean
getProp(task, "missing");              // ❌ "missing" δεν είναι keyof Task
```

Ο compiler ξέρει ότι το `key` είναι έγκυρο κλειδί **και** ότι το αποτέλεσμα έχει τον σωστό τύπο για κάθε κλειδί ξεχωριστά. Αυτό ακριβώς το pattern κρύβεται μέσα στα `Pick`/`Omit`/`Record` που θα δεις στο επόμενο μάθημα.

## `typeof` (type operator) — τύπος από τιμή

Διαφορετικό από το runtime `typeof`. Στο type-level, παίρνει τον **τύπο μιας υπάρχουσας τιμής**:

```ts
const config = { retries: 3, url: "/api" };
type Config = typeof config; // { retries: number; url: string }
```

Συνδυάζεται με `keyof` για να παράγεις τύπους από constants: `type Key = keyof typeof config` → `"retries" | "url"`.

## Πώς απαντάς στη συνέντευξη

> «What are generics, and what do keyof and indexed access types give you?»

> Generics are type parameters — they let me write reusable code that keeps the relationship between input and output instead of collapsing to `any`. A `first<T>(items: T[]): T` returns exactly the element type. Constraints with `extends` restrict what a generic accepts so I can safely use properties inside — `<T extends { id: string }>` guarantees an `id`. `keyof` turns an object type into a union of its keys, and indexed access `T[K]` gets the type of a property by its key. Put together — `getProp<T, K extends keyof T>(obj: T, key: K): T[K]` — I get a fully type-safe getter where the compiler both rejects invalid keys and knows the exact return type per key. That same combination is what powers the built-in utility types like Pick and Record.
