# Next.js App Router + TypeScript

Next.js είναι πολύ συχνό σε fullstack αγγελίες γιατί ενώνει frontend, routing, server rendering, data loading και deployment.

Στο App Router πρέπει να καταλαβαίνεις:

- file-based routing,
- `layout.tsx` και `page.tsx`,
- Server Components και Client Components,
- metadata,
- typed data loading,
- dynamic routes.

## Pages και layouts

Στο App Router κάθε route χτίζεται με folders.

```txt
app/
  projects/
    page.tsx
  layout.tsx
```

Το `layout.tsx` τυλίγει τις σελίδες. Το `page.tsx` είναι το actual route UI.

## Server Components

Στο App Router, components είναι Server Components by default. Αυτό σημαίνει ότι μπορείς να κάνεις server-side data loading χωρίς να στείλεις όλη τη λογική στον browser.

```tsx
export default async function Page() {
  const data = await getData();
  return <main>{data.title}</main>;
}
```

## Client Components

Αν χρειάζεσαι state, effects ή browser APIs, βάζεις:

```tsx
"use client";
```

Μην το βάζεις παντού. Χρησιμοποίησέ το μόνο όταν χρειάζεται interactivity.

## Metadata

```tsx
export const metadata: Metadata = {
  title: "Projects"
};
```

Καλό για SEO και social previews.
