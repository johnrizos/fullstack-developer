# App Router & File-System Routing

Το Next.js χρησιμοποιεί **file-system routing**: φάκελοι και ειδικά αρχεία μέσα στο `app/` ορίζουν τα routes. Στα interviews εξετάζουν αν ξέρεις τη διαφορά `page` vs `layout`, πώς δουλεύουν τα nested/dynamic routes, και — προσοχή στο Next.js 16 — ότι **τα `params`/`searchParams` είναι πλέον Promises**.

## Τι πρέπει να μπορείς στο τέλος

- να φτιάχνεις routes με `page.tsx` και shared UI με `layout.tsx`,
- να χτίζεις nested routes με φακέλους,
- να ορίζεις dynamic segments `[slug]` και να διαβάζεις `params` (await),
- να διαβάζεις `searchParams` και να ξέρεις ότι κάνει το route dynamic,
- να πλοηγείσαι με `<Link>`.

## page & layout

Ένα **page** είναι UI για ένα route· ένα **layout** είναι shared UI που τυλίγει pages και **διατηρεί state στο navigation** (δεν ξανα-renderάρει).

```tsx
// app/page.tsx → route "/"
export default function Page() {
  return <h1>Hello Next.js!</h1>;
}

// app/layout.tsx → root layout (ΥΠΟΧΡΕΩΤΙΚΟ, με html & body)
export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="el">
      <body>
        <main>{children}</main>
      </body>
    </html>
  );
}
```

Το root layout είναι υποχρεωτικό και πρέπει να περιέχει `<html>` και `<body>`. Τα layouts φωλιάζουν: το root τυλίγει το `app/blog/layout.tsx`, που τυλίγει το `app/blog/page.tsx`.

## Nested routes

```txt
app/
  page.tsx                → /
  blog/
    page.tsx              → /blog
    layout.tsx            → shared UI για όλα τα /blog/*
    [slug]/
      page.tsx            → /blog/:slug
```

**Φάκελοι** ορίζουν URL segments· **αρχεία** (`page`, `layout`) φτιάχνουν το UI. Για να γίνει ένα segment δημόσια προσβάσιμο, χρειάζεται `page.tsx`.

## Dynamic segments — params είναι Promise

Όνομα φακέλου σε αγκύλες `[slug]` φτιάχνει dynamic segment. Στο Next.js 16 το `params` είναι **Promise** — κάνεις `await`:

```tsx
// app/blog/[slug]/page.tsx
export default async function BlogPostPage({
  params,
}: {
  params: Promise<{ slug: string }>;
}) {
  const { slug } = await params; // ← await, ΟΧΙ params.slug απευθείας
  const post = await getPost(slug);
  return <h1>{post.title}</h1>;
}
```

Με `generateStaticParams` προ-παράγεις τα slugs στο build για static generation:

```tsx
export function generateStaticParams() {
  return [{ slug: "hello" }, { slug: "world" }];
}
```

## searchParams → dynamic rendering

```tsx
// app/page.tsx
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>;
}) {
  const { q } = await searchParams;
  // ...
}
```

Η χρήση `searchParams` σε server page κάνει το route **dynamic** (χρειάζεται το incoming request). Σε Client Components διαβάζεις query με το `useSearchParams` hook.

## Link & typed route props

```tsx
import Link from "next/link";

<Link href={`/blog/${post.slug}`}>{post.title}</Link>;
```

Το `<Link>` δίνει prefetching και client-side navigation. Επίσης υπάρχουν global typed helpers (παράγονται σε `next dev`/`build`):

```tsx
export default async function Page(props: PageProps<"/blog/[slug]">) {
  const { slug } = await props.params; // πλήρως typed από το route
  return <h1>{slug}</h1>;
}
```

## Πώς απαντάς στη συνέντευξη

«How does routing work in the Next.js App Router?»

> Routing is file-system based inside the `app` directory: folders define URL segments and special files build the UI. A `page.tsx` makes a segment publicly accessible, and a `layout.tsx` is shared UI that wraps nested pages and preserves state across navigations instead of re-rendering. I nest folders for nested routes and wrap a segment name in square brackets, like `[slug]`, for a dynamic segment. A key Next.js 16 detail is that `params` and `searchParams` are now Promises, so I `await` them — `const { slug } = await params`. Reading `searchParams` opts the route into dynamic rendering because it needs the request, and for prerendering dynamic routes I provide `generateStaticParams`. I navigate with `<Link>` for prefetching and client-side transitions, and I lean on the generated `PageProps`/`LayoutProps` helpers for fully typed route props.
