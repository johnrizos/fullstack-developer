# Layout & Responsive

Τα flexbox και grid που έμαθες σε καθαρό CSS υπάρχουν στο Tailwind ως **utilities** — και το responsive γίνεται με **prefixes** αντί για media queries. Αυτό το μάθημα δείχνει πώς στήνεις layouts και πώς τα κάνεις responsive διαβάζοντας τη συμπεριφορά «σε μία γραμμή».

## Flexbox utilities

Οι ίδιες έννοιες (`display:flex`, `align-items`, `justify-content`) ως classes:

```html
<div class="flex items-center justify-between gap-4">
  <span>Αριστερά</span>
  <span>Δεξιά</span>
</div>
```

- `flex` → `display: flex`
- `items-center` / `items-start` → `align-items`
- `justify-between` / `justify-center` → `justify-content`
- `gap-4` → κενό μεταξύ items (από την κλίμακα)
- `flex-col` → κάθετη διεύθυνση· `flex-wrap` → wrapping

## Grid utilities

```html
<!-- 3 ίσες στήλες με κενό -->
<div class="grid grid-cols-3 gap-4">
  <div>A</div><div>B</div><div>C</div>
</div>
```

- `grid` → `display: grid`
- `grid-cols-3` → 3 ίσες στήλες (`repeat(3, 1fr)`)
- `col-span-2` → ένα item πιάνει 2 στήλες
- `gap-4` → κενό γραμμών/στηλών

## Sizing

```html
<div class="w-full max-w-md h-12 min-h-screen">...</div>
```

`w-full` (100%), `max-w-md` (μέγιστο πλάτος container), `h-12` (ύψος από κλίμακα), `min-h-screen` (≥ ύψος οθόνης). Συνδυάζονται με flex/grid για ολοκληρωμένα layouts.

## Responsive — mobile-first με prefixes

Το Tailwind είναι **mobile-first**: η γυμνή class ισχύει **παντού**, και ένα **breakpoint prefix** την αλλάζει από ένα πλάτος κι άνω.

```html
<!-- 1 στήλη σε mobile, 2 από md (768px), 3 από lg (1024px) -->
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">...</div>
```

| Prefix | Από πλάτος |
|---|---|
| (καμία) | όλα (mobile) |
| `sm:` | ≥ 640px |
| `md:` | ≥ 768px |
| `lg:` | ≥ 1024px |
| `xl:` / `2xl:` | ≥ 1280 / 1536px |

Το κλειδί του mobile-first: ξεκινάς από τη **μικρή** οθόνη (γυμνή class) και **προσθέτεις** για μεγαλύτερες με prefixes. Διαβάζεις όλο το responsive behavior σε μία γραμμή — χωρίς media query blocks αλλού.

## Πώς απαντάς στη συνέντευξη

> «How does responsive design work in Tailwind?»

> Tailwind is mobile-first: the bare utility applies everywhere, and a breakpoint prefix overrides it from that width up. So `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` means one column on mobile, two from 768px, three from 1024px — the whole responsive behaviour reads in a single class string instead of separate media-query blocks. Flexbox and grid are just utilities — `flex items-center justify-between`, `grid grid-cols-3 gap-4` — so I get the layout power of CSS with inline, scannable responsive variants.
