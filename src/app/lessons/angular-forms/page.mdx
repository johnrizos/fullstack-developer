# Forms: Reactive & Template-driven

Η Angular προσφέρει **δύο** προσεγγίσεις για forms: **template-driven** (απλές, στο HTML) και **reactive** (στην class, scalable). Σε πραγματική/enterprise δουλειά κυριαρχούν τα **reactive forms** — και αυτά ρωτιούνται.

## Τι πρέπει να μπορείς στο τέλος

- να χτίζεις reactive form με `FormGroup`/`FormControl`,
- να προσθέτεις validation & error messages,
- να γράφεις custom validator,
- να ξέρεις πότε αρκεί template-driven.

## Template-driven: γρήγορο & απλό

Με `[(ngModel)]` και το `FormsModule` στήνεις φόρμα κατευθείαν στο template — καλό για **μικρές, απλές** φόρμες:

```ts
@Component({
  selector: "app-login",
  standalone: true,
  imports: [FormsModule],
  template: `
    <form #f="ngForm" (ngSubmit)="submit(f)">
      <input name="email" [(ngModel)]="email" required email />
      <button [disabled]="f.invalid">Είσοδος</button>
    </form>
  `,
})
export class LoginComponent {
  email = "";
  submit(form: NgForm) { console.log(form.value); }
}
```

Η λογική ζει στο template (validation attributes σαν `required`, `email`). Καθαρό για λίγα πεδία, αλλά δύσκολο να κλιμακωθεί ή να τεσταριστεί.

## Reactive forms: ο κανόνας για σοβαρές φόρμες

Εδώ ορίζεις το **μοντέλο της φόρμας στην class**, ρητά και typed:

```ts
import { Component, inject } from "@angular/core";
import { FormBuilder, ReactiveFormsModule, Validators } from "@angular/forms";

@Component({
  selector: "app-signup",
  standalone: true,
  imports: [ReactiveFormsModule],
  template: `
    <form [formGroup]="form" (ngSubmit)="submit()">
      <input formControlName="name" />
      @if (form.controls.name.invalid && form.controls.name.touched) {
        <small>Το όνομα είναι υποχρεωτικό</small>
      }

      <input formControlName="email" />
      <button [disabled]="form.invalid">Εγγραφή</button>
    </form>
  `,
})
export class SignupComponent {
  private fb = inject(FormBuilder);

  form = this.fb.group({
    name: ["", [Validators.required, Validators.minLength(2)]],
    email: ["", [Validators.required, Validators.email]],
  });

  submit() {
    if (this.form.valid) console.log(this.form.value);
  }
}
```

Δομικά στοιχεία:

- **`FormControl`** — ένα πεδίο (τιμή + validation + state).
- **`FormGroup`** — ομάδα controls (όλη η φόρμα ή ένα τμήμα).
- **`FormBuilder`** (`fb.group`) — συντομογραφία για να τα φτιάχνεις.
- **`[formGroup]`** + **`formControlName`** — συνδέουν το template με το μοντέλο.

Πλεονεκτήματα: το μοντέλο είναι **κεντρικό, typed, testable** χωρίς DOM, και η validation/state (`valid`, `touched`, `dirty`, `errors`) είναι προσβάσιμη προγραμματιστικά.

## Validation & error messages

Built-in validators: `Validators.required`, `minLength`, `maxLength`, `email`, `pattern`, `min`, `max`. Δείχνεις σφάλματα με βάση το state του control:

```ts
@if (form.controls.email.errors?.["required"] && form.controls.email.touched) {
  <small>Το email είναι υποχρεωτικό</small>
}
@if (form.controls.email.errors?.["email"]) {
  <small>Μη έγκυρο email</small>
}
```

Το `touched` αποφεύγει να δείχνεις σφάλμα πριν καν αγγίξει ο χρήστης το πεδίο — βασικό για καλό UX.

### Custom validator

Όταν δεν φτάνουν οι built-in, γράφεις δική σου function που επιστρέφει `null` (έγκυρο) ή ένα error object:

```ts
import { AbstractControl, ValidationErrors } from "@angular/forms";

function noSpaces(control: AbstractControl): ValidationErrors | null {
  const hasSpace = (control.value ?? "").includes(" ");
  return hasSpace ? { noSpaces: true } : null;
}

// χρήση: username: ["", [Validators.required, noSpaces]]
```

## Πώς απαντάς στη συνέντευξη

> Angular has template-driven forms with `[(ngModel)]` — fine for small forms — and reactive forms, which I default to for anything serious. Reactive forms define the model in the class with `FormGroup` and `FormControl` (via `FormBuilder`), so it's typed, testable without the DOM, and easy to extend dynamically. I attach `Validators` like `required` and `email`, write custom validators that return `null` or an error object, and show messages based on control state — gating on `touched` so errors don't appear before the user interacts.
