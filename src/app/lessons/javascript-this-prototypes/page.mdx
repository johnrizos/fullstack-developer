# this, Prototypes & Classes

Το `this` είναι το σημείο όπου «κόβονται» οι περισσότεροι υποψήφιοι σε JavaScript interviews. Ο λόγος: το `this` **δεν** καθορίζεται από το πού γράφτηκε η function, αλλά από το **πώς καλείται** — το λεγόμενο call-site. Μάθε τους 4 κανόνες και θα απαντάς με σιγουριά.

## Οι 4 κανόνες του this

Όταν βλέπεις ένα function call, έλεγξε με αυτή τη σειρά:

**1. `new` binding** — κλήση με `new`; Το `this` είναι το φρέσκο object:

```js
function User(name) {
  this.name = name;
}
const ada = new User("Ada"); // this → το νέο object
```

**2. Explicit binding** — κλήση με `call`, `apply` ή «κλειδωμένο» με `bind`:

```js
function greet() {
  return `Hi, ${this.name}`;
}

const user = { name: "Ada" };
greet.call(user);  // "Hi, Ada" — this ορίζεται ρητά
const bound = greet.bind(user);
bound();           // "Hi, Ada" — μόνιμα δεμένο, ό,που κι αν κληθεί
```

**3. Implicit binding** — κλήση ως method; Το `this` είναι **το object πριν την τελεία**:

```js
const user = {
  name: "Ada",
  greet() {
    return this.name;
  },
};
user.greet(); // "Ada" — this → user
```

**4. Default binding** — σκέτη κλήση; `undefined` σε strict mode (ή το global object σε sloppy mode):

```js
const fn = user.greet;
fn(); // TypeError σε strict mode — this είναι undefined
```

Η σειρά προτεραιότητας: `new` > explicit (`bind`/`call`/`apply`) > implicit (τελεία) > default.

## Το κλασικό bug: «χαμένο» this

Μόλις **αποσπάσεις** μια method από το object της, το implicit binding χάνεται:

```js
const user = {
  name: "Ada",
  greet() {
    return `Hi, ${this.name}`;
  },
};

user.greet();             // ✅ "Hi, Ada" — κλήση με τελεία
const greet = user.greet;
greet();                  // ❌ this undefined — σκέτη κλήση!

setTimeout(user.greet, 100); // ❌ ίδιο πρόβλημα — το setTimeout
                             // καλεί τη function "σκέτη"
```

Λύσεις:

```js
setTimeout(() => user.greet(), 100);     // 1. wrapper — η κλήση κρατά την τελεία
setTimeout(user.greet.bind(user), 100);  // 2. bind — μόνιμο δέσιμο
```

## Arrow functions: ΔΕΝ έχουν δικό τους this

Οι arrow functions δεν παίζουν με τους 4 κανόνες — **κληρονομούν το `this` από το lexical scope** όπου γράφτηκαν. Αυτό τις κάνει ιδανικές για callbacks:

```js
class Timer {
  constructor() {
    this.seconds = 0;
  }

  start() {
    // Arrow: κληρονομεί το this του start() → το instance ✅
    setInterval(() => {
      this.seconds++;
    }, 1000);
  }

  brokenStart() {
    // Κανονική function: ΔΙΚΟ της this → undefined ❌
    setInterval(function () {
      this.seconds++; // TypeError
    }, 1000);
  }
}
```

Αντίστροφη παγίδα: **μην** χρησιμοποιείς arrow ως object method:

```js
const user = {
  name: "Ada",
  greet: () => `Hi, ${this.name}`, // ❌ this ΔΕΝ είναι το user —
};                                  // είναι το this του εξωτερικού scope
user.greet(); // "Hi, undefined"
```

> Σημείωση: στις arrow functions το `bind`/`call`/`apply` **αγνοείται** για το this — δεν μπορείς να το αλλάξεις.

## Prototype chain — πώς δουλεύει η «κληρονομικότητα»

Κάθε object έχει έναν κρυφό σύνδεσμο (`[[Prototype]]`) προς ένα άλλο object. Όταν ζητάς ένα property που δεν υπάρχει στο object, η JavaScript ψάχνει **πάνω στην αλυσίδα**:

```js
const animal = {
  eats: true,
  describe() {
    return `eats: ${this.eats}`;
  },
};

const rabbit = Object.create(animal); // rabbit → animal
rabbit.jumps = true;

rabbit.jumps;      // true — δικό του property
rabbit.eats;       // true — βρέθηκε στο prototype (animal)
rabbit.describe(); // "eats: true" — method από το prototype,
                   // αλλά this → rabbit (αυτός που κάλεσε!)
rabbit.flies;      // undefined — εξαντλήθηκε η αλυσίδα
```

Όταν φτιάχνεις instances με `new`, οι methods ζουν **μία φορά** στο `prototype` — δεν αντιγράφονται σε κάθε instance:

```js
class User {
  constructor(name) {
    this.name = name; // instance property — μία ανά object
  }
  greet() {
    return this.name; // ζει στο User.prototype — κοινή για όλους
  }
}

const a = new User("Ada");
const b = new User("Grace");
a.greet === b.greet; // true — Η ΙΔΙΑ function, μέσω prototype
```

Αυτό είναι και θέμα μνήμης: 10.000 users = 10.000 `name` strings, αλλά **1** `greet` function.

## Classes = syntax πάνω από prototypes

Το `class` δεν είναι νέο object model — είναι καθαρότερο συντακτικό για το prototype system. Σε συνέντευξη αυτή η πρόταση μετράει πολύ:

```js
class User {
  constructor(name) {
    this.name = name;
  }
  greet() {
    return `Hi, ${this.name}`;
  }
}

// Ισοδύναμο "παλιό" pattern:
function OldUser(name) {
  this.name = name;
}
OldUser.prototype.greet = function () {
  return `Hi, ${this.name}`;
};

typeof User;                 // "function" — οι classes ΕΙΝΑΙ functions!
User.prototype.greet;        // εκεί ζει η method
```

Με `extends` η αλυσίδα μεγαλώνει:

```js
class Admin extends User {
  constructor(name, level) {
    super(name); // υποχρεωτικό ΠΡΙΝ το this σε subclass
    this.level = level;
  }
  greet() {
    return `${super.greet()} (admin level ${this.level})`;
  }
}

const admin = new Admin("Grace", 9);
admin.greet(); // "Hi, Grace (admin level 9)"
// Αναζήτηση: admin → Admin.prototype → User.prototype → Object.prototype
```

## Πώς απαντάς στη συνέντευξη

Αν σε ρωτήσουν «what is `this` in JavaScript?»:

> `this` is determined by how a function is called, not where it's defined. There are four rules: with `new` it's the newly created object; with `call`/`apply`/`bind` it's set explicitly; in a method call it's the object before the dot; otherwise it's undefined in strict mode. Arrow functions are the exception — they inherit `this` lexically from their enclosing scope, which is why they're ideal for callbacks.
