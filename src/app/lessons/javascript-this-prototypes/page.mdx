# this, Prototypes & Classes

Το `this` είναι από τα πιο συχνά σημεία που πέφτουν σε JavaScript interviews. Δεν σημαίνει πάντα “το object που βλέπω δίπλα”. Η τιμή του εξαρτάται από το **πώς καλείται** μια function.

## this σε object method

```js
const user = {
  name: "Ada",
  greet() {
    return this.name;
  }
};
```

Όταν καλείς `user.greet()`, το `this` μέσα στη method δείχνει στο `user`.

## this και function calls

Αν πάρεις τη method και την καλέσεις μόνη της, μπορεί να χάσει το αρχικό object context.

```js
const greet = user.greet;
greet(); // το this δεν δείχνει πια απαραίτητα στο user
```

Αυτό εξηγεί πολλά bugs σε callbacks.

## Prototypes

Στη JavaScript, objects μπορούν να “κληρονομούν” properties και methods από άλλο object μέσω prototype chain.

```js
const user = new User("Ada");
user.greet();
```

Η method `greet` δεν χρειάζεται να αντιγραφεί σε κάθε instance. Υπάρχει στο `User.prototype`.

## Classes

Το `class` είναι πιο καθαρό syntax πάνω από prototypes.

```js
class User {
  constructor(name) {
    this.name = name;
  }

  greet() {
    return this.name;
  }
}
```

Σε συνέντευξη είναι καλό να ξέρεις ότι classes στη JavaScript βασίζονται ακόμα στο prototype system.
