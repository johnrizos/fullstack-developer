# Git & GitHub

Fullstack developer χωρίς Git δεν στέκεται σε ομάδα. Σε συνέντευξη πρέπει να μπορείς να εξηγήσεις πώς δουλεύεις με commits, branches και pull requests.

## Mental model

Το Git είναι local version control. Το GitHub είναι remote platform για collaboration.

Βασική ροή:

```bash
git status
git add .
git commit -m "Add login form"
git push
```

## Commit

Ένα commit είναι snapshot αλλαγών με μήνυμα. Καλό commit:

- είναι μικρό,
- έχει ξεκάθαρο σκοπό,
- δεν ανακατεύει άσχετες αλλαγές.

Παράδειγμα καλού μηνύματος:

```txt
Add validated contact form
```

## Branch

Branch είναι απομονωμένη γραμμή δουλειάς.

```bash
git switch -c feature/contact-form
```

Δουλεύεις σε branch ώστε το main να μένει σταθερό.

## Pull request

Pull request είναι πρόταση αλλαγής. Εκεί γίνονται:

- code review,
- automated checks,
- discussion,
- merge.

## Merge conflicts

Conflict συμβαίνει όταν δύο αλλαγές ακουμπάνε το ίδιο σημείο. Η σωστή απάντηση σε interview δεν είναι “φοβάμαι τα conflicts”, αλλά:

1. Διαβάζω και τις δύο αλλαγές.
2. Κρατάω τη σωστή τελική εκδοχή.
3. Τρέχω tests/build.
4. Κάνω commit το resolved conflict.

## GitHub Actions

Συχνά στις αγγελίες ζητάνε CI/CD. Με GitHub Actions μπορείς να τρέχεις lint/build/tests σε κάθε PR.

```yaml
name: CI
on: [pull_request]
jobs:
  checks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run lint
      - run: npm run build
```

## Interview answer template

Αν σε ρωτήσουν “how do you work with Git in a team?”, απάντησε:

> I create focused branches, make small meaningful commits, open pull requests for review, rely on CI checks, and keep main stable. If conflicts happen, I resolve them by understanding both changes and rerunning tests.
