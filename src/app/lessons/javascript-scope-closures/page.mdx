# Scope, Hoisting & Closures

Αν θέλεις να απαντάς καλά σε JavaScript συνέντευξη, πρέπει να καταλαβαίνεις πώς η γλώσσα “βλέπει” μεταβλητές.

## Scope

**Scope** είναι το σημείο του κώδικα όπου μια μεταβλητή είναι διαθέσιμη.

```js
function greet() {
  const message = "Hello";
  console.log(message);
}
```

Το `message` υπάρχει μέσα στη function. Έξω από τη function δεν είναι διαθέσιμο.

## Lexical scope

Η JavaScript αποφασίζει από πού μπορείς να διαβάσεις μια μεταβλητή με βάση το πού γράφτηκε ο κώδικας, όχι από το πού καλείται η function.

```js
const name = "Ada";

function sayName() {
  console.log(name);
}
```

Η `sayName` μπορεί να δει το `name`, επειδή γράφτηκε σε scope που έχει πρόσβαση στο outer scope.

## Hoisting

**Hoisting** σημαίνει ότι declarations επεξεργάζονται πριν τρέξει ο κώδικας. Όμως δεν συμπεριφέρονται όλα ίδια.

- Function declarations μπορούν συνήθως να κληθούν πριν τη γραμμή που γράφτηκαν.
- `let` και `const` δεν μπορείς να τα χρησιμοποιήσεις πριν την αρχικοποίησή τους.
- `var` hoist-άρεται, αλλά αυτό συχνά οδηγεί σε μπερδέματα.

Για σύγχρονο κώδικα, προτίμησε `const` και `let`.

## Closures

**Closure** είναι όταν μια inner function θυμάται μεταβλητές από το outer scope, ακόμα και αφού το outer function έχει τελειώσει.

```js
function createCounter() {
  let count = 0;

  return function increase() {
    count = count + 1;
    return count;
  };
}

const increase = createCounter();
increase(); // 1
increase(); // 2
```

Η inner function θυμάται το `count`. Αυτό είναι closure.

> A closure is when a function keeps access to variables from its lexical scope, even after the outer function has finished executing.
