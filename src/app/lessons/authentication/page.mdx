# Authentication & Authorization

Authentication και authorization δεν είναι το ίδιο.

- **Authentication**: ποιος είσαι;
- **Authorization**: τι επιτρέπεται να κάνεις;

## Sessions

Session-based auth κρατά state στον server ή σε session store. Ο browser κρατά session cookie.

Πλεονεκτήματα:

- εύκολο revoke,
- καλό για server-rendered apps,
- τα secrets μένουν server-side.

Ένα secure session cookie συνήθως έχει:

- `HttpOnly`, για να μην το διαβάζει JavaScript,
- `Secure`, για HTTPS,
- `SameSite`, για CSRF protection,
- λογικό expiration.

Το frontend μπορεί να ξέρει ότι ο χρήστης είναι logged in, αλλά δεν χρειάζεται να κρατά το raw session secret.

## JWT

JWT είναι signed token που περιέχει claims.

Προσοχή:

- μην βάζεις ευαίσθητα δεδομένα μέσα,
- χρειάζεσαι strategy για expiration/revocation,
- πρόσεχε πού το αποθηκεύεις.

JWT payload δεν είναι encrypted από μόνο του. Είναι απλά encoded και signed. Άρα αν βάλεις email, role ή user id, υπολόγισε ότι μπορεί να διαβαστεί από όποιον έχει το token.

```json
{
  "sub": "user_123",
  "role": "admin",
  "exp": 1735689600
}
```

Το backend πρέπει πάντα να επαληθεύει signature και expiration.

## OAuth

OAuth χρησιμοποιείται για login με third-party providers όπως Google/GitHub.

Συνέντευξη: μην πεις “OAuth είναι password login”. Είναι delegation/authorization flow.

Στο login με GitHub, η εφαρμογή σου δεν παίρνει το GitHub password του χρήστη. Παίρνει authorization code, το ανταλλάσσει server-side για token και μετά ζητά profile data από τον provider.

## Protected routes

Σε fullstack app πρέπει να προστατεύεις και:

- UI routes,
- API routes,
- database queries.

Μην βασίζεσαι μόνο στο frontend για security.

Παράδειγμα server-side authorization:

```ts
async function getTask(taskId: string, currentUserId: string) {
  const task = await db.task.findUnique({ where: { id: taskId } });

  if (!task || task.userId !== currentUserId) {
    throw new Error("Not allowed");
  }

  return task;
}
```

## Συχνά λάθη

- Κρύβω κουμπί στο frontend αλλά αφήνω ανοιχτό API route.
- Βάζω role στο localStorage και το εμπιστεύομαι.
- Δεν ελέγχω owner id σε database queries.
- Κρατάω tokens χωρίς expiration.
