# Authentication & Authorization

Authentication και authorization δεν είναι το ίδιο.

- **Authentication**: ποιος είσαι;
- **Authorization**: τι επιτρέπεται να κάνεις;

## Sessions

Session-based auth κρατά state στον server ή σε session store. Ο browser κρατά session cookie.

Πλεονεκτήματα:

- εύκολο revoke,
- καλό για server-rendered apps,
- τα secrets μένουν server-side.

## JWT

JWT είναι signed token που περιέχει claims.

Προσοχή:

- μην βάζεις ευαίσθητα δεδομένα μέσα,
- χρειάζεσαι strategy για expiration/revocation,
- πρόσεχε πού το αποθηκεύεις.

## OAuth

OAuth χρησιμοποιείται για login με third-party providers όπως Google/GitHub.

Συνέντευξη: μην πεις “OAuth είναι password login”. Είναι delegation/authorization flow.

## Protected routes

Σε fullstack app πρέπει να προστατεύεις και:

- UI routes,
- API routes,
- database queries.

Μην βασίζεσαι μόνο στο frontend για security.
