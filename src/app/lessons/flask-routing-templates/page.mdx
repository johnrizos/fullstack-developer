# Routing & Jinja2 Templates

Εδώ εμβαθύνεις στο routing του Flask (dynamic URLs, named routes) και μαθαίνεις το **Jinja2**, τον templating engine που χρησιμοποιεί και το Flask και πολλά άλλα Python εργαλεία.

## Dynamic routes

```python
@app.route("/posts/<int:post_id>")
def post_detail(post_id):
    return f"Post {post_id}"

@app.route("/users/<username>")          # default: string
def profile(username):
    return f"Profile: {username}"

@app.route("/files/<path:filepath>")     # επιτρέπει slashes
def serve(filepath):
    return filepath
```

Το `<int:post_id>` είναι **converter**: πιάνει το κομμάτι του URL, το μετατρέπει σε int, και το περνά ως argument. Άλλοι: `string` (default), `path`, `float`, `uuid`.

## url_for — named routes

```python
from flask import url_for, redirect

@app.route("/go")
def go():
    return redirect(url_for("post_detail", post_id=42))   # → /posts/42
```

Το `url_for` φτιάχνει URL από το **όνομα της view function**. Ποτέ hardcoded paths — αν αλλάξει το route, τα links μένουν σωστά. (Δουλεύει και για static: `url_for('static', filename='css/app.css')`.)

## Rendering templates

```python
from flask import render_template

@app.route("/posts")
def post_list():
    posts = [{"id": 1, "title": "Γεια"}, {"id": 2, "title": "Κόσμε"}]
    return render_template("post_list.html", posts=posts)
```

Τα templates ζουν στον φάκελο `templates/`. Τα keyword arguments του `render_template` γίνονται μεταβλητές μέσα στο template.

## Jinja2 — variables & escaping

```html
<!-- templates/post_list.html -->
<h1>{{ title }}</h1>
<p>{{ post.author }}</p>          {# attribute access #}
<p>{{ user.name|upper }}</p>      {# filter #}
```

Το `{{ }}` κάνει **αυτόματο HTML escaping** — XSS-safe by default, όπως το Django. Filters με `|`: `upper`, `truncate`, `default`, `length`.

## Control flow

```html
{% if posts %}
    <ul>
    {% for post in posts %}
        <li><a href="{{ url_for('post_detail', post_id=post.id) }}">{{ post.title }}</a></li>
    {% endfor %}
    </ul>
{% else %}
    <p>Δεν υπάρχουν posts.</p>
{% endif %}
```

Πρόσεξε το `url_for` **μέσα** στο template — έτσι φτιάχνεις links χωρίς hardcoded paths.

## Template inheritance

```html
<!-- templates/base.html -->
<!DOCTYPE html>
<html>
<head><title>{% block title %}MyApp{% endblock %}</title></head>
<body>
    {% include "nav.html" %}
    <main>{% block content %}{% endblock %}</main>
</body>
</html>
```

```html
<!-- templates/post_list.html -->
{% extends "base.html" %}
{% block title %}Posts{% endblock %}
{% block content %}
    <h1>Όλα τα posts</h1>
{% endblock %}
```

Ίδιο μοτίβο με το Django (επειδή και το Django template language εμπνεύστηκε από το Jinja): `extends` / `block` / `include` για DRY layouts.

## Static files

```html
<link rel="stylesheet" href="{{ url_for('static', filename='css/app.css') }}">
```

Τα static αρχεία (CSS/JS/images) μπαίνουν στον φάκελο `static/` και σερβίρονται αυτόματα στο dev.

## Πώς απαντάς στη συνέντευξη

«How do routing and templating work in Flask?»

> Routes use `@app.route` with converters like `<int:post_id>` to capture and type URL parameters. I never hardcode paths — `url_for('view_name', **params)` builds URLs from the view function's name, in both Python and templates, so links survive route changes. Templating is Jinja2: `render_template` passes context, `{{ }}` auto-escapes for XSS safety, filters format values, and I keep layouts DRY with `extends`/`block`/`include` — the same pattern Django borrowed. Static assets live in `static/` and are referenced via `url_for('static', ...)`.
