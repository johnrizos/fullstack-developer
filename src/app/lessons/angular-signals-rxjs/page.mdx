# Signals & RxJS

Η σύγχρονη Angular έχει **δύο** μοντέλα reactivity: τα νέα **signals** (απλά, synchronous) και το **RxJS** (Observables, για ασύγχρονες ροές). Το να ξέρεις πότε ποιο είναι το θέμα-κλειδί στις σημερινές συνεντεύξεις Angular.

## Τι πρέπει να μπορείς στο τέλος

- να χρησιμοποιείς `signal`, `computed`, `effect`,
- να καταλαβαίνεις τι είναι ένα **Observable** και ο `async` pipe,
- να ξεχωρίζεις **πότε signals και πότε RxJS**,
- να αποφεύγεις memory leaks από subscriptions.

## Signals: η νέα reactivity

Ένα **signal** είναι ένα κουτί με τιμή που ειδοποιεί όποιον το διαβάζει όταν αλλάξει. Θυμίζει το `ref` της Vue.

```ts
import { signal, computed, effect } from "@angular/core";

const count = signal(0);          // δημιουργία
console.log(count());             // διάβασμα: το καλείς ως function → 0
count.set(5);                     // γράψιμο
count.update((n) => n + 1);       // ενημέρωση βάσει προηγούμενης τιμής → 6

const double = computed(() => count() * 2); // παράγωγο, cached signal
double();                         // 12

effect(() => {
  console.log(`count is ${count()}`); // τρέχει όταν αλλάζει ό,τι διαβάζει
});
```

- **`signal(x)`** → writable signal· διαβάζεις με `count()`, γράφεις με `.set`/`.update`.
- **`computed(...)`** → παράγωγο signal, **cached**, ξανατρέχει μόνο όταν αλλάξουν οι εξαρτήσεις (σαν Vue `computed`).
- **`effect(...)`** → side effect που ξανατρέχει αυτόματα όταν αλλάξει κάποιο signal που διαβάζει (σαν Vue `watchEffect`).

Σε component:

```ts
@Component({
  selector: "app-counter",
  standalone: true,
  template: `<button (click)="inc()">{{ count() }}</button>`,
})
export class CounterComponent {
  count = signal(0);
  inc() { this.count.update((n) => n + 1); }
}
```

Τα signals είναι **synchronous** και απλά — ιδανικά για **UI state** (counters, toggles, derived values). Ενεργοποιούν επίσης πιο αποδοτικό change detection.

## RxJS: Observables για ασύγχρονες ροές

Το **RxJS** μοντελοποιεί **streams** τιμών στον χρόνο: HTTP responses, events, WebSocket μηνύματα. Ένα **Observable** είναι μια ροή στην οποία **κάνεις subscribe**.

```ts
import { HttpClient } from "@angular/common/http";

const users$ = http.get("/api/users"); // Observable (σύμβαση: $ στο τέλος)

users$.subscribe((users) => {
  console.log(users); // τρέχει όταν φτάσει το response
});
```

Η δύναμη του RxJS είναι οι **operators** που μετασχηματίζουν ροές:

```ts
import { debounceTime, distinctUntilChanged, switchMap, map } from "rxjs";

searchInput$
  .pipe(
    debounceTime(300),            // περίμενε 300ms αδράνειας
    distinctUntilChanged(),       // αγνόησε ίδιες διαδοχικές τιμές
    switchMap((q) => http.get(`/api/search?q=${q}`)) // ακύρωσε παλιό request
  )
  .subscribe((results) => { /* ... */ });
```

Αυτό το pattern (debounce + switchMap) είναι **η** κλασική λύση για search-as-you-type — δύσκολο με σκέτα signals, εύκολο με RxJS.

## Ο `async` pipe: subscribe χωρίς leaks

Αν κάνεις χειροκίνητο `subscribe`, **πρέπει** να κάνεις `unsubscribe` στο `ngOnDestroy`, αλλιώς διαρρέει μνήμη. Ο **`async` pipe** το κάνει αυτόματα στο template:

```ts
@Component({
  selector: "app-users",
  standalone: true,
  imports: [AsyncPipe],
  template: `
    @for (user of users$ | async; track user.id) {
      <li>{{ user.name }}</li>
    }
  `,
})
export class UsersComponent {
  private http = inject(HttpClient);
  users$ = this.http.get<User[]>("/api/users");
}
```

Ο `async` pipe κάνει subscribe όταν εμφανίζεται το template και **unsubscribe αυτόματα** όταν καταστρέφεται — μηδέν χειροκίνητο cleanup. Είναι ο προτιμώμενος τρόπος.

## Πότε signals & πότε RxJS

| Χρήση | Διάλεξε |
|---|---|
| UI state (toggles, counters, derived) | **Signals** |
| Σύγχρονες παράγωγες τιμές | **Signals** (`computed`) |
| HTTP, WebSocket, streams στον χρόνο | **RxJS** |
| Debounce/throttle/combine ροών | **RxJS** operators |
| Search-as-you-type | **RxJS** (debounce + switchMap) |

Σύγχρονη πρακτική: **signals για state, RxJS για async streams** — και τα γεφυρώνεις (`toSignal`/`toObservable`) όταν χρειαστεί.

## Πώς απαντάς στη συνέντευξη

> Modern Angular has two reactivity models. Signals — `signal`, `computed`, `effect` — are synchronous and great for UI state and derived values; you read a signal by calling it, like `count()`. RxJS handles asynchronous streams over time: HTTP, events, search-as-you-type, where `debounceTime` + `distinctUntilChanged` + `switchMap` cancel stale requests naturally. To consume Observables in templates I use the `async` pipe, which subscribes and unsubscribes automatically and prevents leaks. My rule of thumb: signals for state, RxJS for async streams, bridging with `toSignal`/`toObservable` when needed.
