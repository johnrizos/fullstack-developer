# TypeScript: types, interfaces, unions και generics

TypeScript είναι JavaScript με στατικό type checking. Δεν αντικαθιστά τη JavaScript. Σε βοηθά να βρεις λάθη νωρίτερα και να κάνεις τον κώδικα πιο ξεκάθαρο.

Σε fullstack αγγελίες, TypeScript ζητείται συχνά μαζί με React και Node. Για συνέντευξη πρέπει να μπορείς να εξηγήσεις όχι μόνο τι είναι, αλλά και πώς το χρησιμοποιείς σε πραγματικό app.

## Type aliases

Με `type` μπορείς να ορίσεις δικό σου σχήμα ή union.

```ts
type TaskStatus = "todo" | "doing" | "done";
```

Αυτό λέει ότι το status επιτρέπεται να είναι μόνο μία από αυτές τις τιμές.

## Interfaces

Τα interfaces περιγράφουν το σχήμα ενός object.

```ts
interface Task {
  id: string;
  title: string;
  status: TaskStatus;
}
```

Αν ένα object λείπει `title` ή έχει λάθος `status`, το TypeScript θα το πιάσει.

## Optional properties

Το `?` σημαίνει ότι μια property είναι προαιρετική.

```ts
estimateHours?: number;
```

## Generics

Τα generics κάνουν ένα type επαναχρησιμοποιήσιμο.

```ts
interface ApiResponse<T> {
  data: T;
  error?: string;
}
```

Έτσι μπορείς να έχεις `ApiResponse<Task[]>`, `ApiResponse<User>` κ.λπ.

## Interview answer template

Αν σε ρωτήσουν “why TypeScript?”, απάντησε:

> TypeScript catches type-related mistakes before runtime, documents object shapes, improves refactoring, and makes API contracts clearer across frontend and backend.

Αν σε ρωτήσουν “type vs interface;”, πες:

> Both can describe object shapes. Interfaces are commonly used for object contracts and can be extended or merged. Type aliases are more flexible for unions, primitives, tuples, and complex compositions.
