# Server Actions & Mutations

Τα **Server Actions** σου επιτρέπουν να αλλάζεις δεδομένα στον server **χωρίς να γράψεις χειροκίνητα API endpoint** — τα καλείς απευθείας από forms ή event handlers. Στα interviews εξετάζουν τι είναι το `'use server'`, πώς δείχνεις pending state, και πώς κάνεις revalidate/redirect μετά τη mutation.

## Τι πρέπει να μπορείς στο τέλος

- να ορίζεις Server Functions/Actions με `'use server'`,
- να τις καλείς από `<form action={...}>`,
- να δείχνεις pending state με `useActionState`,
- να χειρίζεσαι expected errors ως return values,
- να κάνεις `revalidatePath`/`revalidateTag` και `redirect`.

## Τι είναι Server Action

Είναι async function που τρέχει **στον server** και την καλείς από τον client μέσω network request (γι' αυτό async). Σε context φόρμας/mutation λέγεται Server Action. Από κάτω χρησιμοποιεί `POST`.

```ts
// app/actions.ts
"use server";
import { auth } from "@/lib/auth";

export async function createPost(formData: FormData) {
  const session = await auth();
  if (!session?.user) throw new Error("Unauthorized"); // ΠΑΝΤΑ auth check

  const title = formData.get("title");
  // mutate data + revalidate cache
}
```

> ⚠️ Τα Server Actions είναι προσβάσιμα με απευθείας POST, όχι μόνο μέσω του UI σου. **Πάντα** κάνε authentication/authorization μέσα σε κάθε action.

## Κλήση από form

Το React επεκτείνει το `<form>` ώστε να δέχεται Server Action στο `action` prop. Η function παίρνει αυτόματα το `FormData`:

```tsx
import { createPost } from "@/app/actions";

export function Form() {
  return (
    <form action={createPost}>
      <input name="title" />
      <button type="submit">Create</button>
    </form>
  );
}
```

Σε Server Components αυτό δουλεύει με **progressive enhancement** — η φόρμα υποβάλλεται ακόμα κι αν δεν έχει φορτώσει η JavaScript.

## Pending state & expected errors με useActionState

Για loading indicator και error μηνύματα, χρησιμοποιείς το `useActionState`. **Τα expected errors τα μοντελοποιείς ως return values**, όχι throw:

```ts
// app/actions.ts
"use server";
export async function createPost(prevState: { message: string }, formData: FormData) {
  const res = await fetch("https://api.example.com/posts", {
    method: "POST",
    body: JSON.stringify({ title: formData.get("title") }),
  });
  if (!res.ok) return { message: "Failed to create post" }; // return, όχι throw
  return { message: "" };
}
```

```tsx
// app/ui/form.tsx
"use client";
import { useActionState } from "react";
import { createPost } from "@/app/actions";

export function Form() {
  const [state, formAction, pending] = useActionState(createPost, { message: "" });
  return (
    <form action={formAction}>
      <input name="title" required />
      {state.message && <p aria-live="polite">{state.message}</p>}
      <button disabled={pending}>{pending ? "Saving…" : "Create Post"}</button>
    </form>
  );
}
```

## Revalidate & redirect μετά τη mutation

Μετά την αλλαγή δεδομένων, ανανεώνεις το cache ώστε το UI να δείξει τα νέα δεδομένα:

```ts
"use server";
import { revalidatePath } from "next/cache";
import { redirect } from "next/navigation";

export async function createPost(formData: FormData) {
  // ... mutate
  revalidatePath("/posts");  // ανανεώνει το cache του /posts
  redirect("/posts");        // πηγαίνει στη σελίδα (throw control-flow — ό,τι είναι μετά ΔΕΝ τρέχει)
}
```

- `revalidatePath(path)` / `revalidateTag(tag)` — λήγει στοχευμένα το cache.
- `updateTag(tag)` — άμεση λήξη tagged data· `refresh()` — ανανεώνει τον client router (όχι tagged data).
- `redirect()` ρίχνει framework exception — βάλε το revalidate **πριν** το redirect.

## Από event handler (Client Component)

```tsx
"use client";
import { incrementLike } from "./actions";
import { useState } from "react";

export default function LikeButton({ initialLikes }: { initialLikes: number }) {
  const [likes, setLikes] = useState(initialLikes);
  return (
    <button onClick={async () => setLikes(await incrementLike())}>
      {likes}
    </button>
  );
}
```

## Πώς απαντάς στη συνέντευξη

«How do mutations work with Server Actions?»

> A Server Action is an async function marked `'use server'` that runs on the server and is callable from the client — from a `<form action={...}>`, which auto-passes `FormData`, or from an event handler. In Server Components forms get progressive enhancement, so they work before JS loads. Because actions are exposed as POST endpoints reachable directly, I authenticate and authorize inside every action, not just in the UI. For UX I use `useActionState` to get a `pending` flag and to surface expected errors — validation or failed requests I model as return values rather than throwing, reserving throws for unexpected exceptions that the `error.tsx` boundary catches. After mutating I call `revalidatePath` or `revalidateTag` to refresh the cache, and if I redirect I do it after revalidating, since `redirect` throws a control-flow exception that stops the rest of the function.
