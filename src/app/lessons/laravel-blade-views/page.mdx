# Blade Templates & Views

Το **Blade** είναι ο templating engine του Laravel. Γράφεις HTML με μικρές, καθαρές οδηγίες — και το Blade κάνει αυτόματα escaping ώστε να είσαι ασφαλής από XSS.

## Echo & escaping

```blade
{{-- escaped: ασφαλές από XSS (αυτόματο htmlspecialchars) --}}
<h1>{{ $title }}</h1>

{{-- raw: ΧΩΡΙΣ escaping — μόνο αν εμπιστεύεσαι 100% το περιεχόμενο --}}
<div>{!! $trustedHtml !!}</div>

{{-- σχόλιο που δεν εμφανίζεται στο HTML --}}
```

Κανόνας: σχεδόν πάντα `{{ }}`. Το `{!! !!}` μόνο για περιεχόμενο που εσύ ελέγχεις (π.χ. sanitized markdown) — αλλιώς ανοίγεις πόρτα σε XSS.

## Control structures

```blade
@if ($posts->isEmpty())
    <p>Δεν υπάρχουν posts.</p>
@else
    @foreach ($posts as $post)
        <li>{{ $post->title }}</li>
    @endforeach
@endif

@forelse ($posts as $post)
    <li>{{ $post->title }}</li>
@empty
    <p>Κενό.</p>
@endforelse
```

Το `@forelse`/`@empty` είναι το βολικό «loop ή μήνυμα κενού» σε ένα block.

## Layouts με inheritance

Φτιάξε ένα master layout και «γέμισέ» το:

```blade
{{-- resources/views/layouts/app.blade.php --}}
<!DOCTYPE html>
<html>
<head><title>@yield('title')</title></head>
<body>
    @include('partials.nav')
    <main>
        @yield('content')
    </main>
</body>
</html>
```

```blade
{{-- resources/views/posts/index.blade.php --}}
@extends('layouts.app')

@section('title', 'Posts')

@section('content')
    <h1>Όλα τα posts</h1>
@endsection
```

- `@yield('x')`: σημείο όπου μπαίνει περιεχόμενο.
- `@section/@extends`: το παιδί γεμίζει τα sections του layout.
- `@include('partials.nav')`: ενσωματώνει ένα κομμάτι (partial).

## Blade Components — ο μοντέρνος τρόπος

```bash
php artisan make:component Alert
```

```blade
{{-- resources/views/components/alert.blade.php --}}
<div class="alert alert-{{ $type }}">
    {{ $slot }}
</div>
```

```blade
{{-- χρήση --}}
<x-alert type="error">
    Κάτι πήγε στραβά!
</x-alert>
```

Τα **components** είναι σαν React components για Blade: reusable κομμάτια UI με props (`type`) και `{{ $slot }}` για το περιεχόμενο. Προτιμότερα από `@include` για επαναχρησιμοποιήσιμα UI elements.

## Πέρασμα δεδομένων & helpers

```blade
{{-- δεδομένα από controller: return view('posts.index', ['posts' => $posts]) --}}

{{ $post->title }}
{{ $post->created_at->diffForHumans() }}   {{-- "πριν 2 ώρες" --}}

<a href="{{ route('posts.show', $post) }}">Δες</a>
@csrf   {{-- hidden CSRF token σε φόρμες --}}
@method('PUT')   {{-- για PUT/DELETE μέσω HTML form --}}
```

Το `@csrf` βάζει αυτόματα το hidden token που έμαθες στο PHP security — εδώ είναι μία λέξη.

## Πώς απαντάς στη συνέντευξη

«How does Blade help and how is it safe?»

> Blade is Laravel's templating engine. `{{ $value }}` auto-escapes output through `htmlspecialchars`, so views are XSS-safe by default; I only use `{!! !!}` for content I fully trust. I structure pages with layout inheritance — `@extends`, `@section`, `@yield` — and extract reusable UI into Blade components with props and a `{{ $slot }}`, which feel like React components. Forms get `@csrf` for the token and `@method` to spoof PUT/DELETE. Data comes from the controller via the array passed to `view()`.
