# Routing & State (Vue Router + Pinia)

Μια πραγματική εφαρμογή έχει **πολλές σελίδες** και **κοινό state** που χρειάζονται διαφορετικά components. Η Vue τα λύνει με δύο επίσημες βιβλιοθήκες: **Vue Router** για navigation και **Pinia** για state management.

## Τι πρέπει να μπορείς στο τέλος

- να ορίζεις routes και να πλοηγείσαι (`<RouterLink>`, `useRouter`),
- να διαβάζεις route params,
- να προστατεύεις routes με **navigation guards**,
- να φτιάχνεις **Pinia store** για shared state.

## Vue Router: οι σελίδες της εφαρμογής

Ορίζεις έναν χάρτη από paths σε components:

```js
// router.js
import { createRouter, createWebHistory } from "vue-router";

const routes = [
  { path: "/", component: () => import("./views/Home.vue") },
  { path: "/users/:id", component: () => import("./views/User.vue") },
  { path: "/:pathMatch(.*)*", component: () => import("./views/NotFound.vue") },
];

export const router = createRouter({
  history: createWebHistory(),
  routes,
});
```

Το `() => import(...)` κάνει **lazy loading**: ο κώδικας κάθε route κατεβαίνει μόνο όταν χρειαστεί — βασικό για performance. Το `<RouterView />` στο App.vue δείχνει το component του τρέχοντος route.

Πλοήγηση με links ή προγραμματιστικά:

```vue
<template>
  <RouterLink to="/">Αρχική</RouterLink>
  <RouterLink :to="`/users/${user.id}`">Προφίλ</RouterLink>
</template>

<script setup>
import { useRouter } from "vue-router";
const router = useRouter();
function goHome() {
  router.push("/"); // προγραμματιστική πλοήγηση
}
</script>
```

### Route params

Διαβάζεις δυναμικά κομμάτια του URL με `useRoute`:

```vue
<script setup>
import { useRoute } from "vue-router";
const route = useRoute();
const userId = route.params.id; // από το /users/:id
</script>
```

> Προσοχή: αν πλοηγείσαι από `/users/1` σε `/users/2`, το component **επαναχρησιμοποιείται** (δεν ξαναδημιουργείται). Παρακολούθησε το `route.params.id` με `watch` για να ξαναφορτώσεις δεδομένα.

## Navigation guards: προστασία routes

Για να μπλοκάρεις πρόσβαση (π.χ. χωρίς login), χρησιμοποιείς guards:

```js
router.beforeEach((to, from) => {
  const isLoggedIn = !!localStorage.getItem("token");
  if (to.path.startsWith("/admin") && !isLoggedIn) {
    return "/login"; // redirect — ακυρώνει την αρχική πλοήγηση
  }
  // return true ή τίποτα → επιτρέπεται
});
```

Ο `beforeEach` τρέχει **πριν** από κάθε navigation. Επιστρέφεις `false`/μια διαδρομή για να μπλοκάρεις/ανακατευθύνεις, ή `true`/τίποτα για να επιτρέψεις.

## Pinia: το επίσημο state management

Όταν πολλά components χρειάζονται **το ίδιο** state (auth user, καλάθι, ρυθμίσεις), το `provide`/`inject` δεν φτάνει. Η **Pinia** δίνει κεντρικά, reactive **stores**:

```js
// stores/cart.js
import { defineStore } from "pinia";
import { ref, computed } from "vue";

export const useCartStore = defineStore("cart", () => {
  const items = ref([]);                                  // state
  const total = computed(() =>                            // getter
    items.value.reduce((s, i) => s + i.price, 0)
  );
  function add(item) { items.value.push(item); }          // action
  return { items, total, add };
});
```

Χρήση σε οποιοδήποτε component:

```vue
<script setup>
import { useCartStore } from "@/stores/cart";
const cart = useCartStore();
cart.add({ name: "Laptop", price: 1200 });
</script>

<template>
  <p>Σύνολο: {{ cart.total }}€ ({{ cart.items.length }} προϊόντα)</p>
</template>
```

Το store είναι **singleton & reactive**: όποιο component το χρησιμοποιεί βλέπει τις ίδιες, ζωντανές τιμές. Δομή: **state** (`ref`), **getters** (`computed`), **actions** (functions). Είναι ουσιαστικά ένα «καθολικό composable».

## Πώς απαντάς στη συνέντευξη

> For navigation I use Vue Router: routes map paths to components, I lazy-load each with a dynamic import for code splitting, navigate with `<RouterLink>` or `router.push`, and read dynamic segments via `useRoute().params`. I protect routes with `beforeEach` guards that redirect when, say, a user isn't authenticated. For shared state across many components I use Pinia — stores with state, getters, and actions that are singleton and reactive, so everyone sees the same live data. I keep `provide`/`inject` for local subtree concerns like theme, and Pinia for true app-wide state like auth or cart.
