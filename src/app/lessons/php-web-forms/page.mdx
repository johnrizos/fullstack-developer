# Web Requests, Forms & Sessions

Εδώ η PHP κάνει αυτό για το οποίο φτιάχτηκε: απαντά σε HTTP requests. Καταλαβαίνεις πώς φτάνουν τα δεδομένα μιας φόρμας στον server και πώς κρατάς τον χρήστη «συνδεδεμένο» μεταξύ requests.

## Το request lifecycle

Κάθε φορά που έρχεται request, η PHP:

1. ξεκινά καθαρά (τίποτα δεν μένει από προηγούμενο request — *shared-nothing*),
2. τρέχει το script σου,
3. στέλνει την απάντηση,
4. καθαρίζει τα πάντα.

Γι' αυτό χρειάζονται sessions/βάση για να «θυμάται» κάτι — η μνήμη χάνεται σε κάθε request.

## Superglobals

Η PHP δίνει τα δεδομένα του request σε ειδικά global arrays:

```php
<?php
$_GET;     // query string params (?q=php)
$_POST;    // δεδομένα φόρμας (method="post")
$_SERVER;  // headers, method, IP, path
$_SESSION; // δεδομένα που επιβιώνουν μεταξύ requests
$_COOKIE;  // cookies του browser
$_FILES;   // uploaded αρχεία
```

## Διάβασε μια φόρμα — με ασφάλεια

```html
<form method="post" action="/register.php">
  <input name="email" type="email">
  <input name="age" type="number">
  <button>Εγγραφή</button>
</form>
```

```php
<?php
declare(strict_types=1);

// ΠΟΤΕ μην εμπιστεύεσαι το input — να ελέγχεις ότι υπάρχει και να το καθαρίζεις
$email = trim($_POST["email"] ?? "");
$age = filter_input(INPUT_POST, "age", FILTER_VALIDATE_INT);

if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
    http_response_code(422);
    exit("Μη έγκυρο email");
}
if ($age === false || $age < 18) {
    http_response_code(422);
    exit("Μη έγκυρη ηλικία");
}
```

Δύο κανόνες χρυσοί:

- Χρησιμοποίησε `?? ""` ώστε να μη σκάει αν λείπει το πεδίο.
- **Validate** με `filter_var`/`filter_input` πριν χρησιμοποιήσεις οτιδήποτε.

## Output escaping (αποφυγή XSS)

Όταν τυπώνεις δεδομένα χρήστη σε HTML, πάντα κάνε escape:

```php
<?php
$name = $_POST["name"] ?? "";
echo "Γεια " . htmlspecialchars($name, ENT_QUOTES, "UTF-8");
```

Χωρίς `htmlspecialchars`, αν ο χρήστης γράψει `<script>...`, εκτελείται στον browser άλλου χρήστη (XSS). Το escaping μετατρέπει `<` σε `&lt;`.

## Cookies

```php
<?php
// set: τιμή που κρατά ο browser και ξαναστέλνει
setcookie("theme", "dark", [
    "expires" => time() + 86400 * 30,
    "httponly" => true,   // δεν το διαβάζει JavaScript (αντι-XSS)
    "samesite" => "Lax",  // αντι-CSRF
]);

// read (στο επόμενο request)
$theme = $_COOKIE["theme"] ?? "light";
```

## Sessions — «θυμήσου» τον χρήστη

```php
<?php
session_start();   // ΠΡΕΠΕΙ να καλεστεί πριν από οποιοδήποτε output

// login
$_SESSION["user_id"] = 42;

// σε επόμενο request
if (isset($_SESSION["user_id"])) {
    echo "Καλωσήρθες, χρήστη {$_SESSION['user_id']}";
}

// logout
session_destroy();
```

Πώς δουλεύει: η PHP βάζει ένα cookie με session ID, και κρατά τα `$_SESSION` δεδομένα στον server. Ο χρήστης βλέπει μόνο ένα τυχαίο ID — όχι τα δεδομένα.

## Πώς απαντάς στη συνέντευξη

«How does PHP keep a user logged in across requests?»

> PHP is shared-nothing: every request starts from scratch, so I can't hold state in memory. To keep a user logged in I use sessions — `session_start()` issues a cookie with a random session ID, and the actual data lives server-side in `$_SESSION`. On each request I check `$_SESSION['user_id']`. I always validate incoming data with `filter_var` before using it, escape output with `htmlspecialchars` to prevent XSS, and set cookies `httponly` and `samesite` to harden them.
