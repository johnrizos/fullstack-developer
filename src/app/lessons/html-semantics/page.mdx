# HTML Structure & Semantics

Η HTML δεν είναι μόνο `div` και κείμενο. Τα semantic elements περιγράφουν τον ρόλο κάθε περιοχής της σελίδας.

Όταν γράφεις σωστή δομή:

- ο κώδικας διαβάζεται πιο εύκολα,
- οι μηχανές αναζήτησης καταλαβαίνουν καλύτερα τη σελίδα,
- τα screen readers βοηθούν καλύτερα τους χρήστες.

## Βασικά semantic elements

- `header`: πάνω περιοχή μιας σελίδας ή ενότητας.
- `nav`: πλοήγηση με links.
- `main`: το κύριο περιεχόμενο της σελίδας.
- `section`: θεματική ενότητα.
- `article`: ανεξάρτητο περιεχόμενο, όπως άρθρο ή post.
- `footer`: κάτω περιοχή με βοηθητικές πληροφορίες.

## Landmarks

Elements όπως `header`, `nav`, `main` και `footer` δημιουργούν landmarks. Τα screen readers μπορούν να τα χρησιμοποιήσουν σαν “χάρτη” της σελίδας.

Συνήθως μια σελίδα πρέπει να έχει ένα κύριο `main`.

```html
<body>
  <header>...</header>
  <main>...</main>
  <footer>...</footer>
</body>
```

## Section vs article

Χρησιμοποίησε `article` όταν το περιεχόμενο μπορεί να σταθεί μόνο του, όπως blog post, news item ή forum post.

Χρησιμοποίησε `section` όταν χωρίζεις μια σελίδα σε θεματικές ενότητες.

```html
<article>
  <h1>Οδηγός HTML</h1>
  <section>
    <h2>Semantics</h2>
    <p>...</p>
  </section>
</article>
```

## Heading hierarchy

Τα headings πρέπει να δείχνουν δομή:

- `h1`: κύριος τίτλος σελίδας,
- `h2`: μεγάλες ενότητες,
- `h3`: υποενότητες.

Μην διαλέγεις heading μόνο επειδή σου αρέσει το μέγεθος. Για μέγεθος χρησιμοποίησε CSS.

## Common mistake

Το semantic HTML δεν σημαίνει ότι δεν θα χρησιμοποιήσεις ποτέ `div`. Το `div` είναι fine όταν δεν υπάρχει καλύτερο semantic element. Απλώς μην το χρησιμοποιείς για τα πάντα από συνήθεια.
