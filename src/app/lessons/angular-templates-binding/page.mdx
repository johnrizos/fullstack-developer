# Templates, Binding & Control Flow

Τα Angular templates συνδέουν τα δεδομένα της class με το HTML μέσω **binding**, και ελέγχουν τι εμφανίζεται μέσω του **νέου control flow** (`@if`, `@for`, `@switch`). Εδώ μαθαίνεις όλους τους τύπους binding και πώς να renderάρεις δυναμικό UI.

## Τι πρέπει να μπορείς στο τέλος

- να ξεχωρίζεις τα 4 είδη binding,
- να χρησιμοποιείς το νέο control flow `@if` / `@for` / `@switch`,
- να γράφεις σωστό `track` στο `@for`,
- να εφαρμόζεις κλάσεις/styles δυναμικά.

## Τα 4 είδη binding

Η σύνδεση class ↔ template γίνεται σε τέσσερις μορφές — η σύνταξη δείχνει την **κατεύθυνση**:

```ts
@Component({
  selector: "app-demo",
  standalone: true,
  template: `
    <!-- 1. Interpolation: class → view (κείμενο) -->
    <h1>{{ title }}</h1>

    <!-- 2. Property binding [ ]: class → view (attribute/property) -->
    <img [src]="imageUrl" [alt]="title" />
    <button [disabled]="isLoading">Αποθήκευση</button>

    <!-- 3. Event binding ( ): view → class -->
    <button (click)="save()">Save</button>
    <input (input)="onType($event)" />

    <!-- 4. Two-way binding [( )]: αμφίδρομο -->
    <input [(ngModel)]="name" />
  `,
})
export class DemoComponent {
  title = "Καλημέρα";
  imageUrl = "/logo.png";
  isLoading = false;
  name = "";
  save() {}
  onType(e: Event) {}
}
```

Μνημονικός κανόνας:

- **`{{ }}`** interpolation → κείμενο μέσα σε element.
- **`[prop]`** property binding → «μέσα» (δεδομένα μπαίνουν στο view). «Square brackets = δεδομένα μπαίνουν».
- **`(event)`** event binding → «έξω» (γεγονότα βγαίνουν στην class). «Parentheses = γεγονότα βγαίνουν».
- **`[(ngModel)]`** two-way → «banana in a box», συνδυασμός των δύο.

> Το `[(ngModel)]` χρειάζεται το `FormsModule` στα imports του standalone component.

## Νέο control flow: `@if`, `@for`, `@switch`

Από την Angular 17 το control flow είναι **built-in στο template** (αντικαθιστά τα παλιά `*ngIf`/`*ngFor`):

```ts
template: `
  @if (user) {
    <p>Καλώς ήρθες, {{ user.name }}</p>
  } @else if (loading) {
    <p>Φόρτωση…</p>
  } @else {
    <a routerLink="/login">Σύνδεση</a>
  }

  <ul>
    @for (todo of todos; track todo.id) {
      <li>{{ todo.title }}</li>
    } @empty {
      <li>Καμία εργασία</li>
    }
  </ul>

  @switch (status) {
    @case ("active") { <span>Ενεργό</span> }
    @case ("paused") { <span>Σε παύση</span> }
    @default { <span>Άγνωστο</span> }
  }
`;
```

Πλεονεκτήματα έναντι των παλιών directives: καθαρότερη σύνταξη, καλύτερο type-narrowing, ενσωματωμένο `@empty`, και είναι **πιο γρήγορο**.

### Το `track` είναι υποχρεωτικό

Στο `@for` **πρέπει** να δηλώσεις `track` — μια μοναδική ταυτότητα ανά item (συνήθως `item.id`). Βοηθά την Angular να ξαναχρησιμοποιεί DOM nodes αντί να τα ξαναχτίζει σε κάθε αλλαγή — ίδια ιδέα με τα keys σε React/Vue. Χωρίς σωστό `track`, οι λίστες είναι αργές και «χάνουν» state (π.χ. focus, input τιμές).

## Δυναμικές κλάσεις & styles

```ts
template: `
  <div [class.active]="isActive" [class.error]="hasError">…</div>
  <div [ngClass]="{ active: isActive, error: hasError }">…</div>
  <div [style.color]="isError ? 'red' : 'green'">…</div>
  <div [ngStyle]="{ fontWeight: isBold ? '700' : '400' }">…</div>
`;
```

Το `[class.x]` τοggλάρει μία κλάση· το `[ngClass]` δέχεται object με πολλές. Αντίστοιχα `[style.prop]` και `[ngStyle]`.

## Πώς απαντάς στη συνέντευξη

> Angular has four binding forms: `{{ }}` interpolation and `[prop]` property binding push data into the view, `(event)` sends events out to the class, and `[(ngModel)]` is two-way. For dynamic UI I use the new built-in control flow — `@if`/`@else`, `@for` with a mandatory `track`, and `@switch` — which replaced the old structural directives with cleaner syntax and better performance. The `track` expression must be a stable unique id, the same reasoning as keys elsewhere; using `$index` on a reorderable list loses DOM state and causes wasted re-renders.
