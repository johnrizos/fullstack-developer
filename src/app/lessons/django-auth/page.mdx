# Authentication & Permissions

Το Django έρχεται με πλήρες auth system: users, login/logout, password hashing, permissions και groups — έτοιμα στο `django.contrib.auth`. Δεν χτίζεις τίποτα από το μηδέν.

## Authentication vs authorization

- **Authentication**: *ποιος είσαι;* (login).
- **Authorization / permissions**: *τι επιτρέπεται να κάνεις;*

## Το User model

```python
from django.contrib.auth.models import User

user = User.objects.create_user(username="anna", password="secret")  # hashes αυτόματα
user.check_password("secret")   # True
user.is_authenticated           # στα views
user.is_staff                   # πρόσβαση στο admin
user.is_superuser
```

Το `create_user` κάνει **hash** τον κωδικό αυτόματα (PBKDF2). Ποτέ δεν αποθηκεύεις plaintext.

## Login / logout

```python
from django.contrib.auth import authenticate, login, logout

def login_view(request):
    if request.method == "POST":
        user = authenticate(
            request,
            username=request.POST["username"],
            password=request.POST["password"],
        )
        if user is not None:
            login(request, user)              # ξεκινά session
            return redirect("dashboard")
    return render(request, "registration/login.html")

def logout_view(request):
    logout(request)
    return redirect("login")
```

Το `authenticate` ελέγχει credentials· το `login` ξεκινά το session. Από εκεί, το `request.user` είναι ο συνδεδεμένος χρήστης σε κάθε request.

## Προστασία views

```python
from django.contrib.auth.decorators import login_required, permission_required

@login_required
def dashboard(request):
    return render(request, "dashboard.html")

@permission_required("blog.add_post", raise_exception=True)
def post_create(request):
    ...
```

```python
# class-based views
from django.contrib.auth.mixins import LoginRequiredMixin

class DashboardView(LoginRequiredMixin, ListView):
    ...
```

Το `@login_required` (FBV) ή `LoginRequiredMixin` (CBV) μπλοκάρει μη-συνδεδεμένους και τους στέλνει στο login.

## Permissions & groups

Το Django φτιάχνει **αυτόματα** permissions για κάθε model (`add`, `change`, `delete`, `view`):

```python
user.has_perm("blog.change_post")        # έλεγχος
user.user_permissions.add(permission)    # ανάθεση

# groups: σύνολα permissions (ρόλοι)
from django.contrib.auth.models import Group
editors = Group.objects.create(name="Editors")
editors.permissions.add(...)
user.groups.add(editors)                 # ο χρήστης κληρονομεί τα perms του group
```

Τα **groups** είναι ρόλοι: αντί να δίνεις permissions ένα-ένα σε κάθε χρήστη, τα ομαδοποιείς (Editors, Moderators) και βάζεις χρήστες σε group.

## Στο template

```html
{% if user.is_authenticated %}
    Γεια, {{ user.username }} — <a href="{% url 'logout' %}">Έξοδος</a>
{% else %}
    <a href="{% url 'login' %}">Είσοδος</a>
{% endif %}

{% if perms.blog.add_post %}
    <a href="{% url 'post_create' %}">Νέο post</a>
{% endif %}
```

## Πώς απαντάς στη συνέντευξη

«How does Django handle auth and permissions?»

> Django ships a full auth system in `django.contrib.auth`. `create_user` hashes passwords with PBKDF2, `authenticate` checks credentials and `login` starts the session, so `request.user` is available everywhere. I protect views with `@login_required` / `LoginRequiredMixin` and check authorization with `@permission_required` or `has_perm`. Django auto-creates add/change/delete/view permissions per model, and I组 group them into roles like "Editors" so users inherit permissions from a group instead of being assigned each one individually. Templates gate UI with `user.is_authenticated` and `perms`.
