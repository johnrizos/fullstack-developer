# Composer, Autoloading & Errors

Το **Composer** είναι ο package manager της PHP — ό,τι το npm για το JavaScript. Διαχειρίζεται dependencies και, κυρίως, κάνει **autoloading** ώστε να μη γράφεις ποτέ ξανά χειροκίνητα `require`.

## Ξεκίνημα project

```bash
composer init        # φτιάχνει composer.json διαδραστικά
composer require guzzlehttp/guzzle   # εγκατάσταση πακέτου
composer install     # εγκαθιστά ό,τι λέει το composer.json/lock
```

Δομή που προκύπτει:

```txt
composer.json      ← τα dependencies σου (τι θες)
composer.lock      ← ακριβείς εκδόσεις (τι ακριβώς εγκαταστάθηκε)
vendor/            ← ο κώδικας των πακέτων (μπαίνει σε .gitignore)
vendor/autoload.php
```

Το `composer.lock` το **κάνεις commit** ώστε όλη η ομάδα να έχει ακριβώς τις ίδιες εκδόσεις — όπως το `package-lock.json`.

## PSR-4 Autoloading

Στο `composer.json` δηλώνεις πώς το namespace αντιστοιχεί σε φακέλους:

```json
{
  "autoload": {
    "psr-4": {
      "App\\": "src/"
    }
  }
}
```

Αυτό λέει: «ό,τι ξεκινά με `App\` βρίσκεται στον φάκελο `src/`». Έτσι το `App\Support\Str` → `src/Support/Str.php`.

Μετά από αλλαγή στο autoload τρέχεις:

```bash
composer dump-autoload
```

Και πλέον στο entry point σου:

```php
<?php
require __DIR__ . "/vendor/autoload.php";  // η ΜΟΝΗ χειροκίνητη require

use App\Support\Str;
echo Str::slug("Hello World");  // φορτώνεται αυτόματα
```

Ένα μόνο `require` για τον autoloader — όλες οι classes φορτώνονται **on demand** όταν χρησιμοποιηθούν.

## Τι είναι τα PSR standards

Τα **PSR** (PHP Standard Recommendations) είναι κοινές συμβάσεις ώστε ο κώδικας όλων να ταιριάζει:

- **PSR-1 / PSR-12**: coding style (indentation, ονόματα, braces).
- **PSR-4**: autoloading (namespace ↔ φάκελοι).
- **PSR-3**: interface για loggers.
- **PSR-7**: HTTP messages (requests/responses).

Δεν χρειάζεται να τα αποστηθίσεις — αρκεί να ξέρεις ότι υπάρχουν και ότι τα frameworks τα ακολουθούν.

## Exceptions — διαχείριση σφαλμάτων

Η μοντέρνα PHP χειρίζεται σφάλματα με exceptions, όχι με return codes:

```php
<?php
function findUser(int $id): array {
    $user = /* ... lookup ... */ null;
    if ($user === null) {
        throw new RuntimeException("User $id not found");
    }
    return $user;
}

try {
    $user = findUser(42);
} catch (RuntimeException $e) {
    echo "Σφάλμα: " . $e->getMessage();
} finally {
    // τρέχει πάντα (cleanup), είτε έσκασε είτε όχι
}
```

## Custom & typed exceptions

```php
<?php
class NotFoundException extends RuntimeException {}
class ValidationException extends RuntimeException {}

try {
    throw new NotFoundException("Δεν βρέθηκε");
} catch (NotFoundException $e) {
    http_response_code(404);
} catch (ValidationException $e) {
    http_response_code(422);
}
```

Φτιάχνεις **δικές σου exception classes** ώστε να πιάνεις διαφορετικά είδη σφαλμάτων ξεχωριστά και να αντιδράς ανάλογα (404 vs 422). Πιάνε **συγκεκριμένες** exceptions πριν τις γενικές.

## Πώς απαντάς στη συνέντευξη

«What does Composer do and how does autoloading work?»

> Composer is PHP's dependency manager. It reads `composer.json` for what I need, locks exact versions in `composer.lock`, and installs packages into `vendor/`. Its killer feature is PSR-4 autoloading: I map a namespace prefix like `App\` to a folder like `src/`, and then a single `require vendor/autoload.php` is enough — classes load on demand by mapping their fully-qualified name to a file path. For errors I use exceptions, create typed exception classes so I can catch and translate different failures into the right HTTP status, and always catch specific exceptions before generic ones.
