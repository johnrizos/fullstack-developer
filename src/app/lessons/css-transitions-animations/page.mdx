# Transitions, Animations & Pseudo-classes

Το «ζωντανό» κομμάτι του CSS: micro-interactions (hover, focus), smooth transitions και keyframe animations. Καλά καμωμένα, κάνουν ένα UI να νιώθει επαγγελματικό· κακώς καμωμένα, κολλάνε. Αυτό το μάθημα καλύπτει pseudo-classes/elements, `transition`, `transform`, `@keyframes` animation, και — κρίσιμο — **ποιες properties είναι φθηνές να κινηθούν** (performance).

## Pseudo-classes & pseudo-elements

**Pseudo-classes** στοχεύουν μια **κατάσταση** (ένα `:`):

```css
a:hover { }          /* ποντίκι από πάνω */
button:focus { }     /* focus (keyboard/click) — μην το σβήνεις, a11y! */
input:disabled { }
li:nth-child(odd) { } /* structural: μονά παιδιά (zebra striping) */
.item:first-child { }
input:checked + label { } /* checkbox/radio state */
```

**Pseudo-elements** φτιάχνουν «εικονικά» elements (δύο `::`):

```css
.quote::before { content: "«"; }   /* εισάγει περιεχόμενο */
.quote::after  { content: "»"; }
p::first-line { font-weight: bold; }
```

## transition — ομαλή αλλαγή τιμής

Το `transition` κάνει μια αλλαγή property να συμβεί **σταδιακά** αντί ακαριαία:

```css
.btn {
  background: #0070f3;
  transition: background 0.2s ease, transform 0.2s ease;
}
.btn:hover {
  background: #0051a8;
  transform: translateY(-2px); /* ανασηκώνεται απαλά */
}
```

Σύνταξη: `transition: <property> <duration> <timing-function> <delay>`. Timing functions: `ease`, `linear`, `ease-in-out`, ή `cubic-bezier(...)`. **Tip**: δήλωσε το `transition` στην **κανονική** κατάσταση (όχι στο `:hover`), ώστε να ισχύει και στο «μπες» και στο «βγες».

## transform — μετακίνηση/κλίμακα/περιστροφή

```css
transform: translateX(20px) scale(1.1) rotate(5deg);
```

Το `transform` (και το `opacity`) είναι **φθηνά** να κινηθούν — ο browser τα χειρίζεται στη GPU χωρίς να ξαναϋπολογίζει layout. Προτίμησέ τα για animations.

## @keyframes — animations πολλαπλών βημάτων

Όταν θες κάτι που τρέχει μόνο του ή έχει στάδια, ορίζεις `@keyframes` και τα εφαρμόζεις με `animation`:

```css
@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}
.spinner {
  animation: spin 1s linear infinite;
}

@keyframes fadeInUp {
  0%   { opacity: 0; transform: translateY(10px); }
  100% { opacity: 1; transform: translateY(0); }
}
```

Σύνταξη: `animation: <name> <duration> <timing> <delay> <iteration-count> <direction>`. `infinite` για ατέρμονο, `forwards` για να κρατήσει την τελική κατάσταση.

## Performance: τι είναι φθηνό να κινείς

Ο κανόνας που ρωτιέται: **κίνησε `transform` και `opacity`**, απόφυγε να κινείς `width`, `height`, `top`, `left`, `margin` σε animations. Τα τελευταία προκαλούν **layout/reflow** (ο browser ξαναϋπολογίζει θέσεις) και **paint** σε κάθε frame → κολλήματα. Τα `transform`/`opacity` τρέχουν στο compositor (GPU) ομαλά.

> Σεβασμός χρήστη: τύλιξε τα έντονα animations σε `@media (prefers-reduced-motion: reduce)` ώστε να τα μειώνεις για όσους έχουν ευαισθησία στην κίνηση.

## Πώς απαντάς στη συνέντευξη

> «How do you add smooth UI animations without hurting performance?»

> For state changes like hover and focus I use `transition` declared on the element's normal state — not on `:hover` — so it animates both in and out, and I animate `transform` and `opacity` rather than layout properties. That's the performance key: `transform` and `opacity` run on the compositor/GPU, while animating `width`, `height`, `top`, or `left` forces layout reflow and paint every frame, which janks. For multi-step or looping motion I define `@keyframes` and apply them with `animation`. I keep focus states visible for accessibility, and wrap heavy motion in `prefers-reduced-motion` so users who opt out get a calmer experience. Pseudo-classes target state (`:hover`, `:focus`, `:nth-child`) and pseudo-elements like `::before`/`::after` add decorative content.
