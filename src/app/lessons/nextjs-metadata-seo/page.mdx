# Metadata, SEO & OG Images

Το Next.js παράγει αυτόματα τα σωστά `<head>` tags από ένα **Metadata API** — static object ή dynamic `generateMetadata`. Στα interviews εξετάζουν πώς κάνεις per-page SEO με δεδομένα, πώς αποφεύγεις διπλά fetches, και πώς φτιάχνεις dynamic OG images.

## Τι πρέπει να μπορείς στο τέλος

- να ορίζεις static `metadata` σε page/layout,
- να φτιάχνεις dynamic `generateMetadata` με δεδομένα,
- να αποφεύγεις διπλό fetch με `React.cache`,
- να παράγεις dynamic OG images με `ImageResponse`,
- να ξέρεις τα file conventions (sitemap, robots, favicon).

## Static metadata

Export ενός `Metadata` object από `layout.tsx` ή `page.tsx`. Το Next.js φτιάχνει τα `<head>` tags:

```tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "My Blog",
  description: "Σκέψεις για web development.",
  openGraph: { title: "My Blog", images: ["/og.png"] },
};

export default function Layout({ children }: { children: React.ReactNode }) {
  return <section>{children}</section>;
}
```

> `metadata` και `generateMetadata` υποστηρίζονται **μόνο σε Server Components**.

## Dynamic metadata: generateMetadata

Όταν το metadata εξαρτάται από δεδομένα (π.χ. τίτλος blog post), χρησιμοποιείς `generateMetadata`. Το `params` είναι **Promise**:

```tsx
// app/blog/[slug]/page.tsx
import type { Metadata } from "next";

export async function generateMetadata({
  params,
}: {
  params: Promise<{ slug: string }>;
}): Promise<Metadata> {
  const { slug } = await params;
  const post = await getPost(slug);
  return { title: post.title, description: post.excerpt };
}

export default async function Page({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params;
  const post = await getPost(slug);
  return <Article post={post} />;
}
```

## Αποφυγή διπλού fetch με React.cache

Παρατήρησε: το `getPost(slug)` καλείται **δύο φορές** — στο `generateMetadata` και στο `Page`. Με `React.cache`, εκτελείται μόνο μία φορά ανά request:

```ts
// app/lib/data.ts
import { cache } from "react";

export const getPost = cache(async (slug: string) => {
  return db.query.posts.findFirst({ where: eq(posts.slug, slug) });
});
```

## Streaming metadata

Για dynamically rendered pages, το Next.js **streamάρει** το metadata ξεχωριστά: το εισάγει στο HTML μόλις resolve-άρει το `generateMetadata`, χωρίς να μπλοκάρει το UI. Για bots/crawlers (Twitterbot, Slackbot, Bingbot) το streaming **απενεργοποιείται** ώστε να βρουν το metadata στο `<head>`.

## Dynamic OG images: ImageResponse

Φτιάχνεις OG image με JSX/CSS μέσω `ImageResponse` από `next/og`, σε αρχείο `opengraph-image.tsx`:

```tsx
// app/blog/[slug]/opengraph-image.tsx
import { ImageResponse } from "next/og";
import { getPost } from "@/app/lib/data";

export const size = { width: 1200, height: 630 };
export const contentType = "image/png";

export default async function Image({ params }: { params: { slug: string } }) {
  const post = await getPost(params.slug);
  return new ImageResponse(
    (
      <div style={{ fontSize: 64, width: "100%", height: "100%", display: "flex",
        alignItems: "center", justifyContent: "center", background: "white" }}>
        {post.title}
      </div>
    ),
    size,
  );
}
```

> `ImageResponse` υποστηρίζει flexbox και ένα υποσύνολο CSS (όχι `display: grid`). Μετατρέπει HTML/CSS σε PNG.

## File-based metadata

```txt
favicon.ico / icon.png / apple-icon.png   → icons
opengraph-image.jpg / twitter-image.jpg   → social images (static ή generated)
robots.txt  (ή robots.ts)                 → crawling rules
sitemap.xml (ή sitemap.ts)                → sitemap
```

Πιο συγκεκριμένο αρχείο (βαθύτερα στο δέντρο) υπερισχύει του γενικότερου.

## Πώς απαντάς στη συνέντευξη

«How do you handle metadata and SEO in Next.js?»

> Next.js generates the `<head>` tags from a Metadata API, only in Server Components. For static pages I export a `metadata` object from the page or layout; when the metadata depends on data — a blog post's title — I export an async `generateMetadata`, awaiting `params` since it's a Promise, and returning the `Metadata`. Because the page itself usually needs the same data, I wrap the fetch in `React.cache` so it runs once per request instead of twice. For dynamically rendered pages metadata streams in separately so it doesn't block the UI, except for crawlers where it stays in the head. For social previews I generate dynamic OG images with `ImageResponse` from `next/og` in an `opengraph-image.tsx`, using JSX and flexbox CSS, and I rely on file conventions like `sitemap.ts`, `robots.ts`, and `favicon.ico`.
