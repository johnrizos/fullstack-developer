# Routing, Controllers & Middleware

Τα routes είναι η «πόρτα» κάθε Laravel εφαρμογής: αντιστοιχίζουν ένα URL + HTTP method σε κώδικα. Εδώ μαθαίνεις routes, controllers και το middleware pipeline.

## Βασικά routes

```php
<?php
// routes/web.php
use App\Http\Controllers\PostController;

Route::get("/posts", [PostController::class, "index"]);
Route::get("/posts/{post}", [PostController::class, "show"]);
Route::post("/posts", [PostController::class, "store"]);
Route::put("/posts/{post}", [PostController::class, "update"]);
Route::delete("/posts/{post}", [PostController::class, "destroy"]);
```

Κάθε HTTP method έχει το δικό του helper. Το `{post}` είναι **route parameter** — δυναμικό κομμάτι του URL.

## Route parameters

```php
<?php
Route::get("/posts/{id}", function (string $id) {
    return "Post $id";
});

// optional με default
Route::get("/users/{name?}", function (?string $name = "guest") {
    return "Γεια $name";
});

// με περιορισμό (μόνο αριθμοί)
Route::get("/posts/{id}", fn(int $id) => "...")->whereNumber("id");
```

## Named routes

```php
<?php
Route::get("/posts/{post}", [PostController::class, "show"])->name("posts.show");

// φτιάξε URL από το όνομα — αν αλλάξει το path, δεν σπάει τίποτα
$url = route("posts.show", ["post" => 42]);   // /posts/42
```

Πάντα δίνε **όνομα** στα routes και αναφέρσου σε αυτά με `route()` αντί να γράφεις hardcoded URLs.

## Resource controllers

Αντί να γράφεις 7 routes με το χέρι:

```php
<?php
Route::resource("posts", PostController::class);
```

```bash
php artisan make:controller PostController --resource
```

Αυτό φτιάχνει αυτόματα τα standard CRUD routes/methods: `index`, `create`, `store`, `show`, `edit`, `update`, `destroy`. Δες τα με `php artisan route:list`.

## Route model binding — το «κλικ»

Αντί να ψάχνεις χειροκίνητα το model από το id:

```php
<?php
// ΧΩΡΙΣ binding
public function show(string $id) {
    $post = Post::findOrFail($id);   // χειροκίνητο lookup
    return view("posts.show", compact("post"));
}

// ΜΕ binding: το {post} γίνεται αυτόματα Post instance
public function show(Post $post) {   // ίδιο όνομα με το {post}
    return view("posts.show", compact("post"));
}
```

Το Laravel βλέπει το type hint `Post` και το όνομα `{post}`, ψάχνει αυτόματα στη βάση, και αν δεν βρει επιστρέφει **404**. Λιγότερος κώδικας, λιγότερα bugs.

## Middleware — το pipeline

Το middleware είναι «φίλτρα» που τρέχουν **πριν/μετά** τον controller — auth, CSRF, logging, rate limiting.

```php
<?php
// εφάρμοσε σε route
Route::get("/dashboard", [DashboardController::class, "index"])
    ->middleware("auth");

// ή σε group
Route::middleware(["auth", "verified"])->group(function () {
    Route::get("/dashboard", ...);
    Route::get("/settings", ...);
});
```

```bash
php artisan make:middleware EnsureUserIsAdmin
```

```php
<?php
public function handle(Request $request, Closure $next): Response {
    if (! $request->user()?->is_admin) {
        abort(403);
    }
    return $next($request);   // πέρνα στο επόμενο middleware/controller
}
```

Σκέψου το σαν αλυσίδα: κάθε middleware ή σταματά το request (`abort`) ή το προωθεί με `$next($request)`.

## Πώς απαντάς στη συνέντευξη

«What is route model binding and middleware?»

> Route model binding lets me type-hint a model in a controller method whose name matches the route parameter — Laravel automatically queries it by id and returns 404 if it's missing, so I skip the manual `findOrFail`. Middleware is the request pipeline: filters that run before or after the controller, like `auth`, CSRF and rate limiting. Each middleware either short-circuits the request with `abort` or calls `$next($request)` to pass it along. I group related routes under shared middleware and always name my routes so I generate URLs with `route()` instead of hardcoding paths.
