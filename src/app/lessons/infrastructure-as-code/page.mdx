# Infrastructure as Code (IaC)

«Πώς στήθηκε αυτός ο server;» Αν η απάντηση είναι «κάποιος μπήκε στο dashboard και πάτησε κουμπιά πέρυσι», έχεις πρόβλημα: κανείς δεν ξέρει την ακριβή κατάσταση, δεν αναπαράγεται, και κάθε αλλαγή είναι ρίσκο. **Infrastructure as Code** σημαίνει ότι περιγράφεις την υποδομή σου (servers, networks, databases, DNS) σε **αρχεία κώδικα** που μπαίνουν σε version control και εφαρμόζονται αυτόματα. Αυτό το μάθημα καλύπτει το **γιατί** (reproducibility), τις βασικές έννοιες (declarative, state, idempotency) και τα συνηθισμένα λάθη.

## Το πρόβλημα: «click-ops» & configuration drift

Όταν στήνεις υποδομή χειροκίνητα από το web console:
- **Δεν αναπαράγεται** — δεν μπορείς να ξαναφτιάξεις πανομοιότυπο περιβάλλον (staging ≠ production).
- **Configuration drift** — η πραγματική κατάσταση αποκλίνει σιγά-σιγά από αυτό που νομίζεις, με μη-καταγεγραμμένες χειροκίνητες αλλαγές.
- **Κανένα audit trail / review** — δεν ξέρεις ποιος άλλαξε τι, πότε και γιατί.
- **Bus factor** — η γνώση είναι στο κεφάλι ενός ατόμου.

## Η λύση: υποδομή ως κώδικας

Με IaC (π.χ. **Terraform**, Pulumi, CloudFormation) γράφεις σε αρχεία **τι** υποδομή θες. Αυτά τα αρχεία:
- μπαίνουν σε **git** → version control, history, code review, PRs,
- εφαρμόζονται **αυτόματα & επαναλήψιμα** σε οποιοδήποτε περιβάλλον,
- είναι η **single source of truth** για το πώς μοιάζει η υποδομή.

## Declarative vs imperative

- **Imperative** (scripts): γράφεις τα **βήματα** («create VM, then attach disk, then…»). Πρέπει να σκεφτείς κάθε μετάβαση.
- **Declarative** (IaC/Terraform): δηλώνεις την **τελική κατάσταση** («θέλω 3 VMs αυτού του τύπου, αυτό το network»). Το εργαλείο υπολογίζει **μόνο του** τι αλλαγές χρειάζονται για να φτάσεις εκεί.

Το declarative κερδίζει: ίδιο μοτίβο με το Kubernetes — δηλώνεις desired state, το εργαλείο κάνει το reconcile.

## State — πώς ξέρει το εργαλείο τι υπάρχει

Το Terraform κρατά ένα αρχείο **state** που χαρτογραφεί «τι έχω δηλώσει» ↔ «τι υπάρχει πραγματικά στο cloud». Κάθε φορά συγκρίνει:

```txt
desired (ο κώδικάς σου) vs state (τι υπάρχει) → plan (τι θα αλλάξει) → apply
```

Το `plan` σού δείχνει **πριν** εφαρμόσεις τι ακριβώς θα δημιουργηθεί/αλλάξει/διαγραφεί — κρίσιμο για ασφάλεια. Το state είναι ευαίσθητο (μπορεί να περιέχει secrets) και **κοινό** στην ομάδα → αποθηκεύεται remote (π.χ. S3) με **locking** ώστε δύο άτομα να μην κάνουν apply ταυτόχρονα.

## Idempotency — το κλειδί

IaC είναι **idempotent**: τρέχεις το ίδιο config πολλές φορές → το αποτέλεσμα είναι το ίδιο. Αν η υποδομή είναι ήδη στην επιθυμητή κατάσταση, ένα `apply` **δεν κάνει τίποτα**. Δεν δημιουργεί διπλά resources. Αυτό το ξεχωρίζει από ένα αφελές «create» script που θα έφτιαχνε δεύτερο VM κάθε φορά που τρέχει.

## Immutable infrastructure

Συχνό μοτίβο με IaC: αντί να **τροποποιείς** έναν υπάρχοντα server (patch, ssh, αλλαγές με το χέρι → drift), **τον αντικαθιστάς** με νέο από το επικαιροποιημένο config. Ίδια φιλοσοφία με τα containers: το server γίνεται «αναλώσιμο», αναπαράξιμο artifact, όχι χειροποίητο «pet».

## Πώς απαντάς στη συνέντευξη

> «What is Infrastructure as Code and why use it?»

> IaC means I describe my infrastructure — servers, networks, databases, DNS — in code files that live in version control and get applied automatically, instead of clicking around a cloud console. The problem with manual setup is it's not reproducible, it drifts from what you think is deployed, and there's no audit trail or review. With a tool like Terraform I write declarative config — I state the desired end state and the tool figures out what changes are needed, same model as Kubernetes. It keeps a state file mapping my config to real resources, so `plan` shows me exactly what will be created, changed, or destroyed before I apply — that's my safety gate. It's idempotent: applying the same config repeatedly converges to the same result, it won't create duplicates. State lives remotely with locking so the team doesn't clobber each other. And it pairs with immutable infrastructure — replace servers from updated config rather than patching them in place, which kills drift.
