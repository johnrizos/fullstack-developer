# Building APIs

Πολύ συχνά το Laravel δεν σερβίρει HTML αλλά **JSON** για ένα React/mobile frontend. Εδώ μαθαίνεις API routes, καθαρά JSON responses με Resources, και token authentication με Sanctum.

## API routes

```php
<?php
// routes/api.php  — αυτόματο prefix /api, stateless, χωρίς CSRF/sessions
use App\Http\Controllers\Api\PostController;

Route::apiResource("posts", PostController::class);
```

Το `apiResource` είναι σαν το `resource` αλλά **χωρίς** τα `create`/`edit` (που είναι για HTML φόρμες) — μένουν index/store/show/update/destroy.

## JSON responses

```php
<?php
public function show(Post $post) {
    return response()->json($post);          // αυτόματα JSON + σωστά headers
}

public function store(Request $request) {
    $post = Post::create($request->validated());
    return response()->json($post, 201);     // 201 Created
}

public function destroy(Post $post) {
    $post->delete();
    return response()->noContent();          // 204
}
```

Επέστρεψε **σωστά HTTP status codes**: 200 ok, 201 created, 204 no content, 404 not found, 422 validation, 401/403 auth.

## API Resources — έλεγξε το σχήμα του JSON

Μην επιστρέφεις ωμά το model (μπορεί να διαρρεύσει πεδία όπως `password`). Χρησιμοποίησε **Resource**:

```bash
php artisan make:resource PostResource
```

```php
<?php
class PostResource extends JsonResource {
    public function toArray(Request $request): array {
        return [
            "id" => $this->id,
            "title" => $this->title,
            "author" => $this->user->name,
            "created_at" => $this->created_at->toIso8601String(),
            // ΔΕΝ εκθέτουμε internal πεδία
        ];
    }
}
```

```php
<?php
public function show(Post $post) {
    return new PostResource($post);
}

public function index() {
    return PostResource::collection(Post::with("user")->paginate(15));
}
```

Το Resource είναι ένα **transformation layer**: εσύ ελέγχεις ακριβώς τι βγαίνει στο JSON, ανεξάρτητα από το schema της βάσης.

## Pagination

```php
<?php
Post::paginate(15);   // επιστρέφει data + meta (current_page, total, links)
```

Το Laravel βάζει αυτόματα `meta`/`links` στο JSON για paginated αποτελέσματα — ο client ξέρει πόσες σελίδες υπάρχουν.

## Token authentication με Sanctum

```php
<?php
// login endpoint: δώσε token
public function login(Request $request) {
    $request->validate(["email" => "required|email", "password" => "required"]);

    if (! Auth::attempt($request->only("email", "password"))) {
        return response()->json(["message" => "Invalid credentials"], 401);
    }

    $token = $request->user()->createToken("api")->plainTextToken;
    return response()->json(["token" => $token]);
}
```

```php
<?php
// protected routes
Route::middleware("auth:sanctum")->group(function () {
    Route::get("/user", fn(Request $r) => $r->user());
    Route::apiResource("posts", PostController::class);
});
```

Ο client στέλνει σε κάθε request: `Authorization: Bearer <token>`. Stateless — κανένα session.

## Versioning & error shape

```php
<?php
// πρόθεμα έκδοσης ώστε να μη σπας παλιούς clients
Route::prefix("v1")->group(function () { /* ... */ });
```

Κράτα **σταθερό σχήμα** για errors σε όλο το API (π.χ. `{ "message": "...", "errors": {...} }`) — ο client βασίζεται σε αυτό. Το Laravel ήδη επιστρέφει 422 με αυτό το σχήμα στα validation errors.

## Πώς απαντάς στη συνέντευξη

«How do you build a clean JSON API in Laravel?»

> I put stateless routes in `api.php` and use `apiResource` for CRUD. Instead of returning raw models — which can leak fields — I shape output with API Resources, a transformation layer where I decide exactly what JSON goes out. I return correct status codes (201, 204, 404, 422) and paginate lists so clients get `meta` and `links`. Auth is token-based with Sanctum: the client sends `Authorization: Bearer <token>`, and I protect routes with `auth:sanctum`. I version the API under a prefix and keep a consistent error shape so clients can rely on it.
