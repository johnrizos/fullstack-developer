# Angular Foundations & Architecture

Η Angular είναι ένα **πλήρες (batteries-included) framework**: σε αντίθεση με τη React/Vue που είναι κυρίως view layer, η Angular φέρνει μαζί router, forms, HTTP, DI και testing. Είναι **TypeScript-first** και πιο δομημένη — γι' αυτό κυριαρχεί σε enterprise. Σε αυτό το track δουλεύουμε με **σύγχρονη Angular (17+)**: standalone components, signals και το νέο control flow.

## Τι πρέπει να μπορείς στο τέλος

- να εξηγείς την αρχιτεκτονική **component / template / service**,
- να δημιουργείς app & components με το **Angular CLI**,
- να γράφεις ένα **standalone component**,
- να καταλαβαίνεις γιατί η Angular είναι TypeScript-first.

## Η αρχιτεκτονική με μια ματιά

Μια Angular εφαρμογή είναι ένα δέντρο από **components**, υποστηριζόμενο από **services**:

- **Component** = μια class με decorator `@Component`, ένα **template** (HTML) και styles. Ελέγχει ένα κομμάτι οθόνης.
- **Template** = το HTML με Angular σύνταξη (bindings, control flow).
- **Service** = class με reusable λογική/δεδομένα, που μοιράζεται μέσω **Dependency Injection** (επόμενα μαθήματα).

```ts
import { Component } from "@angular/core";

@Component({
  selector: "app-hello",
  standalone: true,
  template: `<h1>Γεια, {{ name }}!</h1>`,
})
export class HelloComponent {
  name = "Ada";
}
```

Τρία σημεία:

- **`@Component`** — decorator που «μετατρέπει» την class σε Angular component, με metadata (selector, template, styles).
- **`selector`** — το custom tag (`<app-hello>`) με το οποίο το χρησιμοποιείς.
- **`standalone: true`** — το σύγχρονο μοντέλο: το component δηλώνει μόνο του τα dependencies του, χωρίς τα παλιά **NgModules**.

## Standalone components (το σύγχρονο default)

Παλιά κάθε component έπρεπε να δηλωθεί σε ένα `NgModule`. Από την Angular 17+ τα **standalone components** είναι το default: κάθε component εισάγει **μόνο του** ό,τι χρειάζεται μέσω `imports`.

```ts
import { Component } from "@angular/core";
import { UserCardComponent } from "./user-card.component";

@Component({
  selector: "app-root",
  standalone: true,
  imports: [UserCardComponent], // φέρνει το child component
  template: `<app-user-card />`,
})
export class AppComponent {}
```

Αυτό μειώνει το boilerplate και κάνει τα dependencies κάθε component **ρητά** και τοπικά — πιο εύκολο tree-shaking και κατανόηση.

## Angular CLI: το εργαλείο που τα κάνει όλα

Η Angular έρχεται με ισχυρό CLI που γεννά, τρέχει, τεστάρει και χτίζει:

```bash
npm install -g @angular/cli
ng new my-app          # νέα εφαρμογή (ρωτά για routing, CSS κ.λπ.)
ng serve               # dev server με hot reload
ng generate component user-card   # (ng g c user-card) — φτιάχνει component
ng generate service auth          # φτιάχνει service
ng build                          # production build στο dist/
ng test                           # unit tests
```

Το `ng generate` παράγει αρχεία με τη **σωστή δομή & naming convention** αυτόματα — γι' αυτό η Angular αισθάνεται «οργανωμένη»: όλοι ακολουθούν τις ίδιες συμβάσεις.

## Γιατί TypeScript-first

Η Angular **γράφτηκε σε TypeScript** και το αξιοποιεί παντού: decorators, typed DI, typed forms, typed inputs. Δεν είναι προαιρετικό «extra» όπως αλλού — είναι η γλώσσα του framework.

- Οι **decorators** (`@Component`, `@Injectable`) είναι TS feature.
- Το **DI** βασίζεται σε τύπους για να ξέρει τι να «εγχύσει».
- Τα **types** πιάνουν λάθη στο compile, κρίσιμο σε μεγάλες ομάδες.

## Πώς απαντάς στη συνέντευξη

> Angular is a batteries-included, TypeScript-first framework: components are classes decorated with `@Component` that pair a template with logic, backed by services shared through dependency injection. On modern Angular I use standalone components — they replace NgModules by declaring their own dependencies in an `imports` array, which means less boilerplate and explicit, tree-shakeable deps. The CLI scaffolds everything with consistent conventions, so `ng generate` gives correctly structured components and services, and `ng serve`/`ng build` handle dev and production.
