# Collections & Comprehensions

Εδώ ξεχωρίζει η Python που μοιάζει με μεταφρασμένη JavaScript από την Python που γράφει έμπειρος developer. Το 80% της καθημερινής Python είναι χειρισμός `list`, `dict`, `set`, `tuple` — και ο τρόπος που τα χειρίζεσαι (comprehensions, unpacking, slicing) είναι αυτό που κρίνει αν ο κώδικάς σου είναι Pythonic. Στις συνεντεύξεις, comprehensions και «πότε dict vs list» είναι σχεδόν σίγουρα.

## Τι πρέπει να μπορείς στο τέλος

- να διαλέγεις σωστά ανάμεσα σε list, tuple, dict, set,
- να γράφεις list/dict/set comprehensions αντί για manual loops,
- να κάνεis slicing και unpacking με σιγουριά,
- να εξηγείς γιατί τα tuples είναι immutable και πού το θες,
- να καταλαβαίνεις generators και γιατί εξοικονομούν μνήμη.

## Οι τέσσερις βασικές collections

```py
fruits = ["apple", "banana"]        # list  — ordered, mutable, επιτρέπει διπλά
point = (10, 20)                     # tuple — ordered, IMMUTABLE
user = {"name": "Ana", "age": 30}    # dict  — key→value, ordered (3.7+)
tags = {"py", "web", "py"}           # set   — unordered, unique → {"py", "web"}
```

Πότε τι:
- **list**: σειρά μετράει, αλλάζει, επιτρέπεις διπλά (η default επιλογή).
- **tuple**: σταθερή ομάδα τιμών που δεν αλλάζει — coordinates, μια εγγραφή, return πολλαπλών τιμών. Επειδή είναι immutable, μπορεί να γίνει **dict key** ή **set member**.
- **dict**: lookup με κλειδί, O(1) πρόσβαση — το αντίστοιχο των JS objects/Maps.
- **set**: μοναδικότητα και γρήγορος έλεγχος ύπαρξης (`x in s` είναι O(1)), union/intersection.

## Comprehensions — το σήμα κατατεθέν της Python

Αντί για manual loop με `.append()`, γράφεις τον μετασχηματισμό σε μία γραμμή. Είναι πιο γρήγορο και πιο ευανάγνωστο:

```py
nums = [1, 2, 3, 4, 5]

# JavaScript-στυλ (un-Pythonic):
doubled = []
for n in nums:
    doubled.append(n * 2)

# Pythonic — list comprehension:
doubled = [n * 2 for n in nums]              # [2, 4, 6, 8, 10]

# Με φίλτρο (το αντίστοιχο .filter().map()):
evens_squared = [n ** 2 for n in nums if n % 2 == 0]  # [4, 16]

# Dict comprehension:
lengths = {word: len(word) for word in ["hi", "world"]}  # {'hi': 2, 'world': 5}

# Set comprehension:
initials = {name[0] for name in ["Ana", "Alex", "Bob"]}  # {'A', 'B'}
```

Η αντιστοιχία με JavaScript array methods βοηθά στη συνέντευξη:

```txt
JS:  nums.filter(n => n % 2 === 0).map(n => n ** 2)
PY:  [n ** 2 for n in nums if n % 2 == 0]
```

> Πότε ΟΧΙ comprehension: αν έχει δύο-τρία nested `for` και `if`, γίνεται δυσανάγνωστο — τότε ένα κανονικό loop είναι πιο καθαρό. Η Pythonic αρχή είναι readability, όχι «όλα σε μία γραμμή».

## Slicing — `[start:stop:step]`

```py
items = [0, 1, 2, 3, 4, 5]
items[1:4]    # [1, 2, 3]   — από index 1 έως ΠΡΙΝ το 4
items[:3]     # [0, 1, 2]   — από την αρχή
items[3:]     # [3, 4, 5]   — μέχρι το τέλος
items[::2]    # [0, 2, 4]   — κάθε δεύτερο (step=2)
items[::-1]   # [5,4,3,2,1,0] — αντιστροφή! το πιο γνωστό idiom
text = "hello"
text[::-1]    # 'olleh' — slicing δουλεύει και σε strings
```

Το `items[::-1]` για αντιστροφή είναι κλασική ερώτηση. Slicing **ποτέ δεν σκάει** σε out-of-range — `items[10:20]` δίνει `[]` αντί για error.

## Unpacking — multiple assignment

```py
# Tuple unpacking
x, y = (10, 20)
a, b = b, a                  # swap χωρίς temp variable!

# Star unpacking — μάζεμα του "υπόλοιπου"
first, *rest = [1, 2, 3, 4]  # first=1, rest=[2, 3, 4]
*init, last = [1, 2, 3, 4]   # init=[1, 2, 3], last=4

# Dict unpacking σε function call & merge
defaults = {"role": "member", "active": True}
user = {**defaults, "name": "Ana"}   # merge — σαν το JS spread
```

## Dict — τα idioms που πρέπει να ξέρεις

```py
user = {"name": "Ana"}

user["age"]            # KeyError αν λείπει — σκάει!
user.get("age")        # None αν λείπει — ασφαλές
user.get("age", 0)     # 0 ως default

for key, value in user.items():    # iterate key+value μαζί
    print(key, value)

"name" in user         # έλεγχος ύπαρξης key — O(1)

# Counting pattern με .get ή defaultdict
counts = {}
for ch in "banana":
    counts[ch] = counts.get(ch, 0) + 1   # {'b':1, 'a':3, 'n':2}
```

## Generators — lazy, για μνήμη

Ένα list comprehension με `[]` φτιάχνει **όλη** τη λίστα στη μνήμη. Ένα generator με `()` παράγει τιμές **μία-μία, on demand** — κρίσιμο για μεγάλα/άπειρα streams:

```py
# List — όλα τα 10M στη μνήμη ταυτόχρονα
squares = [n ** 2 for n in range(10_000_000)]

# Generator — καμία τιμή μέχρι να ζητηθεί, σταθερή μνήμη
squares = (n ** 2 for n in range(10_000_000))
total = sum(squares)   # καταναλώνεται καθώς προχωρά

def read_lines(path):
    with open(path) as f:
        for line in f:
            yield line.strip()   # yield = "δώσε μία τιμή, κράτα τη θέση σου"
```

Το `yield` μετατρέπει μια function σε generator: κρατά την κατάστασή της και συνεχίζει από εκεί που σταμάτησε στην επόμενη τιμή. Έτσι διαβάζεις αρχείο 50GB χωρίς να γεμίσεις τη RAM.

## Πώς απαντάς στη συνέντευξη

«How would you remove duplicates from a list and why?»

> The idiomatic way is `list(set(items))` — converting to a set drops duplicates because sets only hold unique values, and membership is O(1). The caveat is that a set is unordered, so if I need to preserve insertion order I'd use `list(dict.fromkeys(items))`, since dicts keep order from Python 3.7 on. I reach for comprehensions over manual loops for transforms, and a generator instead of a list comprehension when the data is large and I only iterate once.
