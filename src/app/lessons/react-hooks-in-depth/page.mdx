# Hooks in Depth

Πέρα από `useState` και `useEffect`, τα υπόλοιπα hooks ξεχωρίζουν τον junior από τον mid/senior. Στα interviews ρωτάνε: «`useRef` vs state;», «πότε `useReducer`;», «τι κάνει το `useCallback`;», «γιατί υπάρχουν οι κανόνες των hooks;».

## Τι πρέπει να μπορείς στο τέλος

- να ξεχωρίζεις `useRef` από state,
- να διαλέγεις `useReducer` όταν το state γίνεται σύνθετο,
- να εξηγείς `useMemo` vs `useCallback`,
- να γράφεις custom hooks που επαναχρησιμοποιούν λογική,
- να εξηγείς γιατί τα hooks καλούνται πάντα top-level.

## Οι κανόνες των hooks — και το γιατί

```tsx
// ✅ πάντα top-level, ίδια σειρά κάθε render
const [a, setA] = useState(0);
const [b, setB] = useState(0);

// ❌ ποτέ μέσα σε if/loop/condition
if (loggedIn) {
  const [name, setName] = useState(""); // σπάει τη σειρά
}
```

Το React αντιστοιχίζει hooks με τη **σειρά κλήσης** τους, όχι με όνομα. Hook μέσα σε `if` αλλάζει τη σειρά μεταξύ renders → το React δίνει το state του ενός hook σε άλλο. Γι' αυτό: top-level, μόνο σε components/custom hooks.

## useRef — τιμή που επιβιώνει χωρίς re-render

Το `useRef` κρατά μια mutable τιμή που **δεν** προκαλεί re-render όταν αλλάζει. Δύο χρήσεις: (1) πρόσβαση σε DOM element, (2) «instance variable» (timer id, προηγούμενη τιμή).

```tsx
const inputRef = useRef<HTMLInputElement>(null);
const renderCount = useRef(0);

renderCount.current += 1; // αλλαγή χωρίς re-render
inputRef.current?.focus(); // DOM access

<input ref={inputRef} />;
```

## useReducer — όταν το state γίνεται σύνθετο

Όταν έχεις πολλά συσχετισμένα κομμάτια state ή σύνθετες μεταβάσεις, το `useReducer` κεντρικοποιεί τη λογική σε μία pure function.

```tsx
type State = { count: number; step: number };
type Action = { type: "inc" } | { type: "dec" } | { type: "setStep"; step: number };

function reducer(state: State, action: Action): State {
  switch (action.type) {
    case "inc": return { ...state, count: state.count + state.step };
    case "dec": return { ...state, count: state.count - state.step };
    case "setStep": return { ...state, step: action.step };
  }
}

const [state, dispatch] = useReducer(reducer, { count: 0, step: 1 });
dispatch({ type: "inc" });
```

Πλεονέκτημα: όλες οι μεταβάσεις σε ένα σημείο, testable χωρίς React (καθαρή function), προβλέψιμες. Διάλεξέ το όταν τα `useState` πολλαπλασιάζονται και αλληλεξαρτώνται.

## useMemo vs useCallback

```tsx
// useMemo: cache-άρει το ΑΠΟΤΕΛΕΣΜΑ ακριβού υπολογισμού
const sorted = useMemo(() => items.slice().sort(compare), [items]);

// useCallback: cache-άρει την ΙΔΙΑ τη function reference
const handleClick = useCallback(() => onSelect(id), [onSelect, id]);
```

`useCallback(fn, deps)` είναι ισοδύναμο με `useMemo(() => fn, deps)`. Χρησιμεύουν (1) για να αποφύγεις ακριβό recompute, (2) για σταθερές references όταν περνάς props σε `React.memo` children ή σε dependency arrays. Δεν τα βάζεις παντού — έχουν δικό τους κόστος.

## Custom hooks — επαναχρησιμοποίηση λογικής

Ένα custom hook είναι μια function που ξεκινά με `use` και καλεί άλλα hooks. Βγάζει stateful λογική έξω από το component.

```tsx
function useToggle(initial = false) {
  const [on, setOn] = useState(initial);
  const toggle = useCallback(() => setOn((v) => !v), []);
  return [on, toggle] as const;
}

function useDebounce<T>(value: T, ms: number) {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const id = setTimeout(() => setDebounced(value), ms);
    return () => clearTimeout(id);
  }, [value, ms]);
  return debounced;
}
```

Τα custom hooks μοιράζονται **λογική**, όχι state — κάθε χρήση έχει το δικό της ανεξάρτητο state.

## Πώς απαντάς στη συνέντευξη

«useRef vs state, and why the rules of hooks?»

> State drives rendering — changing it triggers a re-render. A ref holds a mutable value that survives across renders but does not cause a re-render, so I use it for things the UI doesn't display: a DOM node to focus, a timer id, the previous value. The rules of hooks exist because React identifies each hook by call order, not by name. If a hook sits inside a condition, some renders call it and some don't, the order shifts, and React hands one hook's state to another — that's why hooks must be called unconditionally at the top level. When state becomes several interdependent pieces with complex transitions, I reach for `useReducer` to centralize them in a pure, testable function, and I extract reusable stateful logic into custom hooks, which share logic, not state.
