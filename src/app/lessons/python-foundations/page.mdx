# Python Foundations

Αν έρχεσαι από JavaScript, η Python δεν είναι «άλλη γλώσσα που μαθαίνεις από την αρχή» — είναι οι ίδιες έννοιες (variables, functions, loops, scope) με διαφορετική φιλοσοφία: **explicit, readable, ένας σωστός τρόπος να γίνει το κάθε πράγμα**. Στις συνεντύεξεις για fullstack/backend ρόλους με Python, το φίλτρο δεν είναι «ξέρεις syntax;» αλλά «γράφεις Python που μοιάζει με Python, ή JavaScript μεταφρασμένο;».

## Τι πρέπει να μπορείς στο τέλος

- να διαβάζεις και να γράφεις βασική Python με σωστό indentation και naming (PEP 8),
- να εξηγείς τη διαφορά dynamic typing της Python από το JavaScript coercion,
- να χρησιμοποιείς `if/elif/else`, `for`, `while` και να ξέρεις τι είναι truthy/falsy,
- να γράφεις functions με default args, keyword args, `*args/**kwargs`,
- να καταλαβαίνεις τι είναι ο `if __name__ == "__main__"` guard.

## Variables & τύποι — dynamic αλλά strong

Η Python είναι **dynamically typed** (δεν δηλώνεις τύπο) αλλά **strongly typed** (δεν κάνει σιωπηλά coercion όπως η JavaScript):

```py
x = 5            # int
name = "Giannis" # str
price = 9.99     # float
active = True    # bool (κεφαλαίο T!)
nothing = None   # το ισοδύναμο του null/undefined

# Strong typing: αυτό ΣΚΑΕΙ, δεν γίνεται "55" όπως στη JS
"5" + 5  # TypeError: can only concatenate str (not "int") to str
```

Αυτή είναι η πρώτη μεγάλη διαφορά που πρέπει να αρθρώνεις: στη JavaScript `"5" + 5 === "55"` και `"5" * 2 === 10` — σιωπηλό coercion που κρύβει bugs. Στην Python ένα λάθος τύπου **γίνεται αμέσως exception**, που είναι πιο ασφαλές.

## Indentation αντί για `{}`

Στην Python η **στοίχιση ΕΙΝΑΙ το syntax** — δεν υπάρχουν `{}` για blocks. 4 κενά ανά επίπεδο (PEP 8). Λάθος indentation = `IndentationError`, όχι απλώς άσχημο style:

```py
def greet(name):
    if name:
        print(f"Hello {name}")   # 8 κενά: μέσα στο if, μέσα στο def
    else:
        print("Hello stranger")  # ίδιο επίπεδο με το if
    print("Done")                # 4 κενά: πίσω στο σώμα του def
```

Το `f"..."` είναι **f-string** — interpolation σαν τα template literals της JS (`` `Hello ${name}` ``). Είναι ο σωστός τρόπος για formatting.

## Truthiness — τι θεωρείται falsy

Στα `if` και τα boolean expressions, αυτά είναι **falsy**: `False`, `None`, `0`, `0.0`, `""`, `[]`, `{}`, `()`, `set()`. Όλα τα άλλα είναι truthy. Πρόσεξε: σε αντίθεση με τη JS, **άδειες collections είναι falsy** — πανίσχυρο idiom:

```py
items = []
if items:                # "αν έχει στοιχεία" — Pythonic
    print("has items")
else:
    print("empty")       # τυπώνεται αυτό

# ΟΧΙ έτσι (un-Pythonic, σαν JavaScript):
if len(items) > 0:       # δουλεύει αλλά δεν είναι ο τρόπος
    ...
```

## Functions — default, keyword & variadic args

```py
def create_user(name, role="member", *, active=True):
    return {"name": name, "role": role, "active": active}

create_user("Ana")                      # role="member", active=True
create_user("Bob", "admin")             # positional
create_user("Cleo", role="admin")       # keyword
create_user("Dan", active=False)        # το * επιβάλλει keyword-only για το active
```

Δύο πράγματα-κλειδιά:
- Το `*` μέσα στη λίστα παραμέτρων σημαίνει «ό,τι ακολουθεί είναι **keyword-only**» — ο καλών *πρέπει* να γράψει `active=False`, ποτέ σκέτο `False`. Κάνει το call site αυτο-τεκμηριωμένο.
- `*args` μαζεύει επιπλέον positional args σε tuple, `**kwargs` μαζεύει επιπλέον keyword args σε dict:

```py
def log(level, *args, **kwargs):
    print(level, args, kwargs)

log("INFO", "saved", 42, user="ana", ms=12)
# INFO ('saved', 42) {'user': 'ana', 'ms': 12}
```

> Mutable default trap — από τα πιο κλασικά Python gotchas στη συνέντευξη: **ΠΟΤΕ** μη βάζεις mutable object (`[]`, `{}`) ως default argument. Το default αξιολογείται **μία φορά** στον ορισμό της function, όχι σε κάθε κλήση, οπότε μοιράζεται ανάμεσα σε όλες τις κλήσεις.

## Ο `__main__` guard

Κάθε αρχείο Python είναι και module που μπορεί να γίνει import. Ο guard ξεχωρίζει «τρέχω το αρχείο απευθείας» από «κάποιος με κάνει import»:

```py
def main():
    print("Running directly")

if __name__ == "__main__":
    main()   # τρέχει μόνο με `python file.py`, ΟΧΙ όταν γίνει import
```

Όταν τρέχεις `python file.py`, η Python θέτει `__name__ == "__main__"`. Όταν το κάνεις `import file`, το `__name__` γίνεται `"file"` — έτσι το `main()` δεν τρέχει κατά λάθος στο import. Είναι το standard entry-point pattern κάθε σοβαρού Python script.

## Πώς απαντάς στη συνέντευξη

«Coming from JavaScript, what surprised you about Python?»

> Three things. First, it's strongly typed — `"5" + 5` raises a TypeError instead of silently concatenating, which catches bugs early. Second, whitespace is syntax, not style — indentation defines blocks. Third, the idioms are different: empty collections are falsy, so I write `if items:` not `if len(items) > 0:`, and I use f-strings and comprehensions instead of manual loops. The biggest gotcha I watch for is mutable default arguments, since defaults are evaluated once at definition time.
