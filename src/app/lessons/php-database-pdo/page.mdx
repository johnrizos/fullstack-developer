# Databases with PDO

Σχεδόν κάθε PHP εφαρμογή μιλάει με βάση δεδομένων. Το **PDO** (PHP Data Objects) είναι ο standard, ασφαλής τρόπος — δουλεύει με MySQL, PostgreSQL, SQLite με τον ίδιο κώδικα.

## Σύνδεση

```php
<?php
declare(strict_types=1);

$pdo = new PDO(
    "mysql:host=localhost;dbname=shop;charset=utf8mb4",
    "user",
    "password",
    [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,  // πέτα exceptions σε λάθη
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC, // γραμμές ως associative arrays
        PDO::ATTR_EMULATE_PREPARES => false,          // πραγματικά prepared statements
    ],
);
```

Αυτές οι τρεις επιλογές είναι **must**: exceptions σε σφάλματα, associative fetch, και απενεργοποίηση emulation για γνήσια ασφάλεια.

## Prepared statements — η ασφάλεια

**ΠΟΤΕ** μη βάζεις δεδομένα χρήστη απευθείας σε SQL string. Αυτό είναι SQL injection:

```php
<?php
// ❌ ΕΠΙΚΙΝΔΥΝΟ — ο χρήστης μπορεί να γράψει: ' OR '1'='1
$pdo->query("SELECT * FROM users WHERE email = '$email'");
```

Αντ' αυτού, χρησιμοποίησε **prepared statements** με placeholders:

```php
<?php
// ✅ ΑΣΦΑΛΕΣ — τα δεδομένα στέλνονται ξεχωριστά από το query
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = :email");
$stmt->execute(["email" => $email]);
$user = $stmt->fetch();   // μία γραμμή (ή false)
```

Με placeholders, η βάση δεν μπερδεύει ποτέ τα **δεδομένα** με τις **εντολές** — αυτό σκοτώνει το SQL injection.

## SELECT — fetch modes

```php
<?php
$stmt = $pdo->prepare("SELECT id, name FROM users WHERE active = :active");
$stmt->execute(["active" => 1]);

$all = $stmt->fetchAll();   // πίνακας από όλες τις γραμμές
// ή γραμμή-γραμμή σε loop:
foreach ($stmt as $row) {
    echo $row["name"];
}
```

- `fetch()`: μία γραμμή.
- `fetchAll()`: όλες (προσοχή σε τεράστια αποτελέσματα).
- iterate το statement: γραμμή-γραμμή, οικονομικό σε μνήμη.

## INSERT / UPDATE / DELETE

```php
<?php
$stmt = $pdo->prepare(
    "INSERT INTO users (name, email) VALUES (:name, :email)"
);
$stmt->execute(["name" => "Άννα", "email" => "anna@example.com"]);

$id = $pdo->lastInsertId();   // το id της νέας γραμμής

$upd = $pdo->prepare("UPDATE users SET name = :name WHERE id = :id");
$upd->execute(["name" => "Anna", "id" => $id]);
echo $upd->rowCount();        // πόσες γραμμές επηρεάστηκαν
```

## Transactions — όλα ή τίποτα

Όταν πολλές αλλαγές πρέπει να πετύχουν **μαζί** (π.χ. μεταφορά χρημάτων), τις τυλίγεις σε transaction:

```php
<?php
try {
    $pdo->beginTransaction();

    $pdo->prepare("UPDATE accounts SET balance = balance - 100 WHERE id = 1")->execute();
    $pdo->prepare("UPDATE accounts SET balance = balance + 100 WHERE id = 2")->execute();

    $pdo->commit();           // εφάρμοσε τα πάντα
} catch (Throwable $e) {
    $pdo->rollBack();         // ακύρωσε τα πάντα αν κάτι έσκασε
    throw $e;
}
```

Αν σκάσει η δεύτερη ενημέρωση, το `rollBack()` αναιρεί και την πρώτη — δεν μένεις ποτέ σε ενδιάμεση, λάθος κατάσταση.

## Πώς απαντάς στη συνέντευξη

«How do you prevent SQL injection in PHP?»

> I never concatenate user input into SQL. I use PDO prepared statements with placeholders and pass the values to `execute()`, so the database treats them as data, never as SQL — that's what eliminates injection. I configure PDO to throw exceptions on errors, fetch as associative arrays, and disable prepare emulation. For operations that must succeed together I wrap them in a transaction with `beginTransaction`/`commit` and `rollBack` in a catch, so the database is never left half-updated.
