# Docker & CI/CD

Docker και CI/CD εμφανίζονται συχνά σε αγγελίες γιατί δείχνουν ότι μπορείς να παραδώσεις λογισμικό με προβλέψιμο τρόπο.

## Docker

Docker container πακετάρει app + dependencies.

```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build
CMD ["npm", "start"]
```

Το σημαντικό δεν είναι να αποστηθίσεις Dockerfile. Είναι να καταλάβεις ότι το container δίνει πιο σταθερό runtime από το “στο δικό μου μηχάνημα δουλεύει”.

Ένα καλό Docker setup προσέχει:

- μικρή εικόνα,
- repeatable install με `npm ci`,
- όχι secrets μέσα στο image,
- σωστό `NODE_ENV`,
- health checks όπου χρειάζονται.

## CI

Continuous Integration σημαίνει ότι checks τρέχουν αυτόματα σε κάθε PR.

```yaml
- run: npm ci
- run: npm run lint
- run: npm run build
```

Ένα τυπικό CI pipeline για fullstack app μπορεί να έχει:

- install dependencies,
- lint,
- typecheck,
- unit tests,
- build,
- integration tests,
- artifact ή preview deploy.

Αν κάποιο βήμα αποτύχει, το PR δεν πρέπει να γίνεται merge μέχρι να διορθωθεί.

## CD

Continuous Deployment/Delivery σημαίνει ότι verified αλλαγές μπορούν να πάνε σε environment με προβλέψιμο workflow.

Υπάρχουν δύο συχνά μοντέλα:

- **Continuous Delivery**: το pipeline ετοιμάζει release, αλλά άνθρωπος πατάει deploy.
- **Continuous Deployment**: όταν περάσουν τα checks, το deploy γίνεται αυτόματα.

## Docker Compose

Σε local development, συχνά χρησιμοποιείς Docker Compose για app + database.

```yaml
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"
```

Δεν χρειάζεται να τρέχουν όλα σε Docker από την πρώτη μέρα. Αλλά πρέπει να καταλαβαίνεις πότε βοηθά: ίδια database για όλους, εύκολο reset, predictable setup.

## Pipeline failures

Όταν αποτύχει CI, διάβασε:

1. Ποιο job απέτυχε.
2. Ποιο command απέτυχε.
3. Ποιο error message εμφανίστηκε πρώτο.
4. Αν το failure αναπαράγεται local.

Μην κάνεις bypass checks χωρίς λόγο. Τα checks υπάρχουν για να προστατεύουν το main branch.

## Mini project: GitHub Actions workflow

Φτιάξε ένα `.github/workflows/ci.yml` με βήματα:

```yaml
name: CI

on:
  pull_request:
  push:
    branches: [main]

jobs:
  checks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - run: npm run lint
      - run: npm run build
```

## Τι να προσέχεις

- Μην κάνεις deploy αν δεν περνάει build.
- Μην τυπώνεις secrets στα logs.
- Μην κάνεις deploy migrations χωρίς rollback plan.
- Κράτα τα CI jobs γρήγορα ώστε η ομάδα να τα χρησιμοποιεί πραγματικά.
