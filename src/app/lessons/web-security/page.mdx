# Web Security Basics

Security δεν είναι “θα το δούμε στο τέλος”. Κάθε fullstack developer πρέπει να ξέρει τα βασικά.

## XSS

Cross-site scripting συμβαίνει όταν untrusted input εκτελείται σαν JavaScript στο browser.

Άμυνες:

- escaping/sanitization,
- όχι raw HTML από χρήστη,
- Content Security Policy,
- frameworks που κάνουν escaping by default.

Παράδειγμα κινδύνου:

```html
<div id="comment"></div>
<script>
  comment.innerHTML = userInput;
</script>
```

Αν το `userInput` περιέχει `<script>...</script>`, μπορεί να εκτελεστεί κώδικας. Σε React, το default rendering κάνει escaping, αλλά `dangerouslySetInnerHTML` χρειάζεται πολύ προσοχή και sanitization.

## CSRF

CSRF είναι όταν τρίτο site προσπαθεί να κάνει request με τα cookies του χρήστη.

Άμυνες:

- SameSite cookies,
- CSRF tokens,
- έλεγχος origin/referer σε sensitive actions.

Το CSRF είναι πιο σχετικό όταν χρησιμοποιείς cookies για auth, επειδή ο browser τα στέλνει αυτόματα. Για sensitive actions όπως αλλαγή email ή πληρωμή, θέλεις επιπλέον προστασία.

## Validation

Πρέπει να κάνεις validation server-side. Client-side validation είναι για UX, όχι για security.

Παράδειγμα server-side thinking:

```ts
function validateTaskTitle(title: unknown) {
  if (typeof title !== "string" || title.trim().length < 2) {
    throw new Error("Title must be at least 2 characters");
  }
}
```

Ο client μπορεί να στείλει ό,τι θέλει με curl, Postman ή custom script. Ο server είναι το boundary που πρέπει να προστατεύει τα δεδομένα.

## Secrets

API keys, database URLs, OAuth secrets:

- όχι στο frontend bundle,
- όχι σε Git,
- χρήση environment variables/secret manager.

## Authorization

Security δεν είναι μόνο login. Πρέπει να ελέγχεις αν ο χρήστης έχει δικαίωμα για το συγκεκριμένο resource.

```ts
if (task.userId !== currentUser.id) {
  return Response.json({ error: "Forbidden" }, { status: 403 });
}
```

Χωρίς αυτό, ένας logged-in χρήστης μπορεί να δοκιμάσει IDs άλλων χρηστών.

## Mini project: secure form handling

Πάρε ένα contact form και πρόσθεσε:

1. Server-side validation για όλα τα fields.
2. Escaped rendering για submitted data.
3. Rate limit ή basic spam protection.
4. Error responses χωρίς sensitive details.
5. Logging χωρίς email body ή secrets.

## Security checklist

- Validate input server-side.
- Escape output.
- Keep secrets out of frontend and Git.
- Use secure cookies.
- Enforce authorization on API/database layer.
- Return generic errors για sensitive failures.
