# JavaScript Logic: conditions και αποφάσεις

Τα προγράμματα δεν κάνουν πάντα το ίδιο πράγμα. Παίρνουν αποφάσεις με βάση δεδομένα.

Η πιο βασική δομή απόφασης είναι το `if / else`.

```js
if (score >= 60) {
  console.log("Πέρασες");
} else {
  console.log("Δοκίμασε ξανά");
}
```

Αυτό σημαίνει: αν η συνθήκη είναι αληθινή, τρέξε το πρώτο block. Αλλιώς, τρέξε το δεύτερο.

## Boolean values

Οι συνθήκες καταλήγουν σε `true` ή `false`.

```js
const isPassing = score >= 60;
const hasPerfectScore = score === 100;
```

Στον κώδικα συχνά κρατάμε τέτοιες τιμές σε variables για να διαβάζεται καλύτερα η απόφαση.

## Comparison operators

Συχνά operators:

- `>` μεγαλύτερο από,
- `>=` μεγαλύτερο ή ίσο,
- `<` μικρότερο από,
- `<=` μικρότερο ή ίσο,
- `===` ίσο χωρίς type coercion,
- `!==` διαφορετικό.

Προτίμησε `===` αντί για `==` μέχρι να καταλάβεις καλά το coercion της JavaScript.

## Truthy, falsy & type coercion

Σε ένα `if`, η JavaScript μετατρέπει κάθε τιμή σε boolean. Οι **falsy** τιμές (που μετράνε ως `false`) είναι λίγες — αξίζει να τις μάθεις απ' έξω:

```js
false, 0, "" (κενό string), null, undefined, NaN
```

**Οτιδήποτε άλλο** είναι **truthy** — ακόμα και το `"0"` (string), το `[]` (κενό array) και το `{}` (κενό object). Αυτό προκαλεί κλασικά bugs:

```js
if (items.length) { /* τρέχει μόνο αν length !== 0 — σωστό */ }
if ([]) { /* ΠΑΝΤΑ τρέχει — το κενό array είναι truthy! */ }
```

Το `==` (loose equality) κάνει **type coercion** πριν συγκρίνει, και τα αποτελέσματα συχνά εκπλήσσουν:

```js
"5" == 5        // true  — το string γίνεται number
0 == ""         // true  — και τα δύο γίνονται falsy/0
null == undefined // true
NaN === NaN     // false — το NaN δεν ισούται με τίποτα, ούτε με τον εαυτό του!
```

Δύο παγίδες που πρέπει να ξέρεις:

- **`typeof null === "object"`** — ιστορικό bug της γλώσσας. Για να ελέγξεις `null` χρησιμοποίησε `value === null`.
- **`NaN`** — για να το πιάσεις χρησιμοποίησε `Number.isNaN(x)`, ποτέ `x === NaN`.

Ο κανόνας: **πάντα `===`** (συγκρίνει τιμή ΚΑΙ τύπο, χωρίς εκπλήξεις), και να ξέρεις τις falsy τιμές για να γράφεις σωστά conditions.

## Multiple branches

Αν έχεις περισσότερα από δύο outcomes, μπορείς να χρησιμοποιήσεις `else if`.

```js
if (score >= 90) {
  console.log("Άριστα");
} else if (score >= 60) {
  console.log("Πέρασες");
} else {
  console.log("Δοκίμασε ξανά");
}
```

Η σειρά έχει σημασία. Βάζεις πρώτα το πιο συγκεκριμένο ή πιο αυστηρό condition.

## Ternary operator `? :`

Ο τριαδικός (ternary / conditional) τελεστής είναι ένα `if/else` σε **μία γραμμή που επιστρέφει τιμή**. Σύνταξη: `condition ? ifTrue : ifFalse`.

```js
const status = score >= 60 ? "Πέρασες" : "Δοκίμασε ξανά";
```

Είναι ισοδύναμο με αυτό, αλλά γίνεται **expression** (παράγει τιμή), οπότε μπαίνει εκεί που το `if` δεν χωράει:

```js
let status;
if (score >= 60) {
  status = "Πέρασες";
} else {
  status = "Δοκίμασε ξανά";
}
```

Γι' αυτό τον βλέπεις συνέχεια μέσα σε template literals και σε JSX/React, όπου χρειάζεσαι τιμή — όχι μπλοκ κώδικα:

```js
console.log(`Αποτέλεσμα: ${score >= 60 ? "✅" : "❌"}`);
```

Κανόνας υγιεινής: **μην φωλιάζεις** ternaries (`a ? b : c ? d : e`) — διαβάζονται απαίσια. Αν χρειάζεσαι πολλαπλά outcomes, γύρνα σε `if / else if` ή `switch`. Το ternary είναι για **δύο** καθαρές επιλογές.

## Logical operators

Μπορείς να συνδυάσεις conditions:

```js
if (score >= 60 && attendance >= 80) {
  console.log("Πέρασες το μάθημα");
}

if (role === "admin" || role === "owner") {
  console.log("Έχεις πρόσβαση");
}
```

Το `&&` σημαίνει “και τα δύο”. Το `||` σημαίνει “τουλάχιστον ένα”.

Σημαντικό: τα `&&` και `||` δεν επιστρέφουν απαραίτητα boolean — επιστρέφουν **μία από τις δύο τιμές** με **short-circuit**. Το `||` δίνει την πρώτη truthy (ή την τελευταία), το `&&` την πρώτη falsy (ή την τελευταία):

```js
const name = inputName || "Ανώνυμος"; // αν inputName είναι falsy → "Ανώνυμος"
isLoggedIn && showDashboard();         // καλεί showDashboard ΜΟΝΟ αν isLoggedIn
```

## Optional chaining `?.` & nullish coalescing `??`

Δύο σύγχρονοι operators που θα γράφεις κάθε μέρα — και ρωτιούνται συχνά σε συνεντεύξεις.

**Optional chaining `?.`** — διαβάζεις «βαθιά» χωρίς να σκάσει αν κάτι στη μέση είναι `null`/`undefined`:

```js
const city = user.address.city;   // ❌ TypeError αν δεν υπάρχει address
const city = user?.address?.city; // ✅ undefined αν λείπει κάτι — χωρίς crash
user.save?.();                    // καλεί τη save ΜΟΝΟ αν υπάρχει
```

**Nullish coalescing `??`** — δίνει default τιμή, αλλά **μόνο** για `null`/`undefined`:

```js
const count = settings.count ?? 10; // 10 μόνο αν count είναι null/undefined
```

Η κρίσιμη διαφορά από το `||`: το `||` «πατάει» πάνω σε **κάθε** falsy τιμή — άρα και πάνω σε `0`, `""` και `false`, που συχνά είναι έγκυρες τιμές:

```js
const qty = order.quantity || 1; // ❌ αν quantity === 0, παίρνεις 1 (λάθος!)
const qty = order.quantity ?? 1; // ✅ το 0 περνάει· default μόνο σε null/undefined
```

Υπάρχει και το `??=` που αναθέτει default μόνο αν η μεταβλητή είναι `null`/`undefined`: `config.timeout ??= 5000`.
