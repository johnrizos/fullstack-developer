# Components, Props & Events

Μια Vue εφαρμογή είναι ένα **δέντρο από components**. Το πώς μιλάνε μεταξύ τους — δεδομένα προς τα κάτω, γεγονότα προς τα πάνω — είναι ο πυρήνας κάθε framework. Εδώ μαθαίνεις **props**, **events** και **slots** με το σύγχρονο `<script setup>`.

## Τι πρέπει να μπορείς στο τέλος

- να ορίζεις typed props με `defineProps`,
- να στέλνεις γεγονότα προς τον γονέα με `defineEmits`,
- να περνάς markup μέσα σε component με **slots**,
- να εξηγείς το **one-way data flow** της Vue.

## Props: δεδομένα προς τα κάτω

Ο γονέας περνά δεδομένα στο παιδί μέσω **props**. Στο παιδί τα δηλώνεις με `defineProps`:

```vue
<!-- UserCard.vue (παιδί) -->
<script setup>
const props = defineProps({
  name: { type: String, required: true },
  role: { type: String, default: "member" },
});
</script>

<template>
  <article>
    <h3>{{ name }}</h3>
    <span>{{ role }}</span>
  </article>
</template>
```

Ο γονέας τα δίνει σαν attributes (με `:` αν είναι δυναμικά):

```vue
<template>
  <UserCard name="Ada" role="admin" />
  <UserCard :name="currentUser.name" />
</template>
```

Με TypeScript η σύνταξη γίνεται πιο καθαρή — δηλώνεις τους τύπους generic:

```vue
<script setup lang="ts">
const props = defineProps<{
  name: string;
  role?: string;
}>();
</script>
```

> **Props are read-only.** Το παιδί **δεν** αλλάζει props — αυτό είναι one-way data flow, ίδιο concept με τη React. Αν χρειαστείς αλλαγή, στέλνεις event στον γονέα.

## Events: γεγονότα προς τα πάνω

Το παιδί ειδοποιεί τον γονέα εκπέμποντας **custom events** με `defineEmits`:

```vue
<!-- TodoItem.vue (παιδί) -->
<script setup>
const props = defineProps<{ id: number; title: string }>();
const emit = defineEmits<{
  remove: [id: number];
  toggle: [id: number];
}>();
</script>

<template>
  <li>
    {{ title }}
    <button @click="emit('toggle', id)">✓</button>
    <button @click="emit('remove', id)">✕</button>
  </li>
</template>
```

Ο γονέας ακούει με `@event`:

```vue
<template>
  <TodoItem
    v-for="todo in todos"
    :key="todo.id"
    :id="todo.id"
    :title="todo.title"
    @remove="removeTodo"
    @toggle="toggleTodo"
  />
</template>
```

Το pattern είναι σταθερό: **props down, events up**. Ο γονέας κατέχει το state· το παιδί ζητά αλλαγές μέσω events.

## Slots: περνάς markup, όχι μόνο δεδομένα

Τα **slots** επιτρέπουν στον γονέα να βάλει **περιεχόμενο** μέσα στο παιδί — σαν τα `children` στη React:

```vue
<!-- Card.vue -->
<template>
  <div class="card">
    <header><slot name="header" /></header>
    <div class="body"><slot /></div>
  </div>
</template>
```

```vue
<!-- χρήση -->
<Card>
  <template #header><h3>Τίτλος</h3></template>
  <p>Αυτό μπαίνει στο default slot.</p>
</Card>
```

- **Default slot** (`<slot />`) → το περιεχόμενο χωρίς όνομα.
- **Named slots** (`<slot name="header" />` + `<template #header>`) → πολλαπλά «σημεία ένθεσης».
- **Scoped slots** → το παιδί μπορεί να **περάσει δεδομένα πίσω** στο slot content (`<slot :item="x" />`), χρήσιμο για λίστες/πίνακες.

Τα slots είναι το βασικό εργαλείο **composition** στη Vue: φτιάχνεις γενικά «κουτιά» (Card, Modal, List) και ο γονέας γεμίζει το περιεχόμενο.

## Πώς απαντάς στη συνέντευξη

> Vue components communicate with props down and events up. I declare typed props with `defineProps` — they're read-only, so a child never mutates them; instead it emits a custom event via `defineEmits` and the parent, which owns the state, makes the change. For composition I use slots, including named and scoped slots, to build generic containers like Card or Modal that the parent fills with content — conceptually the same role as `children` in React.
