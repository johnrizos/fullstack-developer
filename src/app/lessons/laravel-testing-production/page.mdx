# Testing, Queues & Production

Το τελευταίο κομμάτι για να γίνει μια Laravel εφαρμογή production-ready: αυτοματοποιημένα tests, background jobs για αργές εργασίες, caching, και σωστό deployment.

## Testing με Pest/PHPUnit

Το Laravel έρχεται με testing setup. Το **Pest** δίνει καθαρότερη σύνταξη πάνω από το PHPUnit:

```php
<?php
// tests/Feature/PostTest.php
use App\Models\User;

it("creates a post", function () {
    $user = User::factory()->create();

    $response = $this->actingAs($user)->post("/posts", [
        "title" => "Γεια",
        "body" => "Περιεχόμενο",
    ]);

    $response->assertRedirect();
    $this->assertDatabaseHas("posts", ["title" => "Γεια"]);
});
```

```bash
php artisan test
```

- **Feature tests**: χτυπούν πραγματικά routes (request → response → DB) — η μεγαλύτερη αξία.
- **Unit tests**: μεμονωμένες classes/methods.

## Factories & RefreshDatabase

```php
<?php
// δημιουργία test δεδομένων
User::factory()->count(3)->create();
Post::factory()->for($user)->create();
```

Το trait `RefreshDatabase` καθαρίζει τη βάση πριν από κάθε test, ώστε τα tests να είναι ανεξάρτητα μεταξύ τους.

## Queues & Jobs — βγάλε την αργή δουλειά απ' το request

Αργές εργασίες (email, εικόνες, εξωτερικά APIs) δεν πρέπει να κρατάνε τον χρήστη να περιμένει. Τις βάζεις σε **queue**:

```bash
php artisan make:job SendWelcomeEmail
```

```php
<?php
class SendWelcomeEmail implements ShouldQueue {
    use Queueable;

    public function __construct(public User $user) {}

    public function handle(): void {
        // αργή δουλειά — τρέχει στο παρασκήνιο
        Mail::to($this->user)->send(new WelcomeMail());
    }
}
```

```php
<?php
// dispatch: επιστρέφει ΑΜΕΣΩΣ, η δουλειά γίνεται μετά
SendWelcomeEmail::dispatch($user);
```

```bash
php artisan queue:work   # ο worker που εκτελεί τα jobs
```

Ο χρήστης παίρνει απάντηση αμέσως· το email στέλνεται από έναν worker στο παρασκήνιο. Τα jobs έχουν αυτόματα **retries** σε αποτυχία.

## Caching

```php
<?php
use Illuminate\Support\Facades\Cache;

// "remember": αν υπάρχει στην cache δώσ' το, αλλιώς υπολόγισέ το & αποθήκευσέ το
$stats = Cache::remember("homepage-stats", 600, function () {
    return Post::expensiveAggregate();   // τρέχει μόνο αν λείπει από cache
});
```

Το `Cache::remember` είναι το πιο χρήσιμο pattern: ακριβά queries υπολογίζονται μία φορά και σερβίρονται από cache (Redis/file) για 10 λεπτά.

## Production checklist

```bash
composer install --no-dev --optimize-autoloader
php artisan config:cache      # cache config (γι' αυτό config() όχι env())
php artisan route:cache
php artisan view:cache
php artisan migrate --force
```

Επιπλέον:

- `APP_ENV=production`, `APP_DEBUG=false` (αλλιώς διαρρέεις stack traces).
- HTTPS παντού, secrets στο `.env` του server (όχι στο git).
- Queue worker + scheduler ως **supervised** processes (systemd/Supervisor).
- Logs & monitoring (π.χ. Sentry).

## Πώς απαντάς στη συνέντευξη

«How do you make a Laravel app production-ready?»

> I write feature tests that hit real routes and assert on the database with `assertDatabaseHas`, using factories and `RefreshDatabase` for isolation. Slow work — emails, images, third-party calls — goes into queued jobs dispatched with `dispatch()` and run by `queue:work`, so the user gets an immediate response and failures retry automatically. I cache expensive reads with `Cache::remember`. For deployment I set `APP_DEBUG=false`, run `config:cache`, `route:cache`, `view:cache` and `migrate --force`, serve over HTTPS, keep secrets in the server's `.env`, and run the queue worker and scheduler as supervised processes with logging and monitoring.
