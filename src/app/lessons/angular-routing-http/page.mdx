# Routing & HttpClient

Δύο κομμάτια που έρχονται «μέσα στο κουτί» της Angular: ο **Router** για navigation μεταξύ σελίδων και ο **HttpClient** για επικοινωνία με APIs. Εδώ μαθαίνεις routes, params, lazy loading, guards, HTTP calls και interceptors.

## Τι πρέπει να μπορείς στο τέλος

- να ορίζεις routes με params και **lazy loading**,
- να προστατεύεις routes με **guards**,
- να κάνεις HTTP calls με `HttpClient`,
- να χρησιμοποιείς **interceptors** για cross-cutting λογική.

## Routing: οι σελίδες της εφαρμογής

Ορίζεις τα routes ως array και τα δηλώνεις στο bootstrap:

```ts
// app.routes.ts
import { Routes } from "@angular/router";

export const routes: Routes = [
  { path: "", component: HomeComponent },
  { path: "users/:id", component: UserComponent },
  {
    path: "admin",
    loadComponent: () => import("./admin.component").then((m) => m.AdminComponent),
  },
  { path: "**", component: NotFoundComponent }, // catch-all 404
];
```

Το **`loadComponent`** κάνει **lazy loading**: ο κώδικας του route κατεβαίνει μόνο όταν ο χρήστης πάει εκεί (code splitting, μικρότερο αρχικό bundle). Το `<router-outlet />` στο template δείχνει το component του τρέχοντος route.

Πλοήγηση:

```ts
template: `
  <a routerLink="/">Αρχική</a>
  <a [routerLink]="['/users', user.id]">Προφίλ</a>
`;
```

```ts
import { Router } from "@angular/router";
const router = inject(Router);
router.navigate(["/users", id]); // προγραμματιστικά
```

### Route params

```ts
import { ActivatedRoute } from "@angular/router";

const route = inject(ActivatedRoute);
const id = route.snapshot.paramMap.get("id");       // μία φορά (snapshot)
route.paramMap.subscribe((p) => load(p.get("id"))); // reactive σε αλλαγές
```

> Το `snapshot` είναι ΟΚ αν το component δεν επαναχρησιμοποιείται. Αν πλοηγείσαι `/users/1 → /users/2` (ίδιο component), χρησιμοποίησε το **observable** `paramMap` για να ξαναφορτώσεις.

## Guards: προστασία routes

Ένα **guard** είναι μια function που αποφασίζει αν επιτρέπεται η πλοήγηση. Το σύγχρονο API είναι functional:

```ts
import { CanActivateFn, Router } from "@angular/router";
import { inject } from "@angular/core";

export const authGuard: CanActivateFn = () => {
  const router = inject(Router);
  const loggedIn = !!localStorage.getItem("token");
  return loggedIn ? true : router.createUrlTree(["/login"]);
};

// route: { path: "admin", canActivate: [authGuard], ... }
```

Επιστρέφεις `true` (επιτρέπεται), `false` (μπλοκ), ή ένα `UrlTree` (redirect).

## HttpClient: επικοινωνία με APIs

Ο `HttpClient` επιστρέφει **Observables** (δες το μάθημα Signals & RxJS):

```ts
import { HttpClient } from "@angular/common/http";

@Injectable({ providedIn: "root" })
export class UserApi {
  private http = inject(HttpClient);

  getUsers() {
    return this.http.get<User[]>("/api/users");
  }
  createUser(data: NewUser) {
    return this.http.post<User>("/api/users", data);
  }
}
```

Τον στήνεις στο bootstrap με `provideHttpClient()`. Στο component τον καταναλώνεις είτε με `subscribe` (με cleanup) είτε καλύτερα με τον **`async` pipe** στο template — που κάνει unsubscribe αυτόματα.

> Το `get<User[]>` δίνει **typed** απάντηση — ένα από τα οφέλη του TypeScript-first.

## Interceptors: cross-cutting λογική

Ένας **interceptor** «πιάνει» **κάθε** HTTP request/response για κοινή λογική: auth headers, logging, error handling, retries — σε ένα μέρος αντί σε κάθε call:

```ts
import { HttpInterceptorFn } from "@angular/common/http";

export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const token = localStorage.getItem("token");
  const authReq = token
    ? req.clone({ setHeaders: { Authorization: `Bearer ${token}` } })
    : req;
  return next(authReq);
};

// bootstrap: provideHttpClient(withInterceptors([authInterceptor]))
```

Κλασικές χρήσεις: προσθήκη token σε κάθε request, καθολικό error handling (π.χ. redirect σε login στο 401), logging.

## Πώς απαντάς στη συνέντευξη

> Angular ships routing and HTTP in the box. Routes map paths to components, I lazy-load with `loadComponent` for code splitting, navigate with `routerLink` or `router.navigate`, and read params from `ActivatedRoute` — snapshot for one-off, the `paramMap` observable when the component is reused. I protect routes with functional guards returning `true`, `false`, or a `UrlTree` redirect. For data I use `HttpClient`, which returns typed Observables I consume via the `async` pipe, and I put cross-cutting concerns like auth headers or global error handling in an HTTP interceptor so they live in one place.
