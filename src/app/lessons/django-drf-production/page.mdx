# REST APIs (DRF) & Production

Όταν το frontend είναι React/mobile, το Django σερβίρει **JSON** μέσω του **Django REST Framework (DRF)** — το de-facto standard για APIs στο Django. Κλείνουμε το track με DRF και production setup.

## Setup

```bash
pip install djangorestframework
```

```python
# settings.py
INSTALLED_APPS = [..., "rest_framework"]
```

## Serializers — το σχήμα του JSON

Ο serializer μετατρέπει models ↔ JSON και κάνει validation (το αντίστοιχο των API Resources + Form):

```python
# blog/serializers.py
from rest_framework import serializers
from .models import Post

class PostSerializer(serializers.ModelSerializer):
    author = serializers.ReadOnlyField(source="author.username")

    class Meta:
        model = Post
        fields = ["id", "title", "body", "author", "created_at"]
        read_only_fields = ["author", "created_at"]
```

Ελέγχεις **ακριβώς** ποια πεδία βγαίνουν/μπαίνουν — δεν εκθέτεις internal πεδία.

## Views: APIView vs ViewSets

```python
# function-based με decorator
from rest_framework.decorators import api_view
from rest_framework.response import Response

@api_view(["GET", "POST"])
def post_list(request):
    if request.method == "GET":
        posts = Post.objects.all()
        return Response(PostSerializer(posts, many=True).data)
    serializer = PostSerializer(data=request.data)
    serializer.is_valid(raise_exception=True)
    serializer.save(author=request.user)
    return Response(serializer.data, status=201)
```

```python
# ViewSet: όλο το CRUD με ελάχιστο κώδικα
from rest_framework import viewsets

class PostViewSet(viewsets.ModelViewSet):
    queryset = Post.objects.all()
    serializer_class = PostSerializer
```

```python
# urls.py — router φτιάχνει αυτόματα τα CRUD routes
from rest_framework.routers import DefaultRouter

router = DefaultRouter()
router.register("posts", PostViewSet)
urlpatterns = [path("api/", include(router.urls))]
```

Ένα `ModelViewSet` + router σου δίνει **όλα** τα REST endpoints (list/create/retrieve/update/destroy) με σωστά status codes — το αντίστοιχο του `apiResource`.

## Authentication & permissions

```python
# settings.py
REST_FRAMEWORK = {
    "DEFAULT_AUTHENTICATION_CLASSES": ["rest_framework.authentication.TokenAuthentication"],
    "DEFAULT_PERMISSION_CLASSES": ["rest_framework.permissions.IsAuthenticated"],
}
```

```python
from rest_framework.permissions import IsAuthenticatedOrReadOnly

class PostViewSet(viewsets.ModelViewSet):
    permission_classes = [IsAuthenticatedOrReadOnly]
```

Ο client στέλνει `Authorization: Token <token>`. Το DRF χειρίζεται auth & permissions δηλωτικά, ανά view.

## Production checklist

```python
# settings.py — ξεχώρισε ανά περιβάλλον, secrets από env
import os
DEBUG = os.environ.get("DEBUG") == "1"
SECRET_KEY = os.environ["SECRET_KEY"]
ALLOWED_HOSTS = os.environ.get("ALLOWED_HOSTS", "").split(",")
```

```bash
python manage.py collectstatic   # μάζεψε static files
python manage.py migrate
gunicorn config.wsgi             # production WSGI server (όχι runserver!)
```

Production essentials:

- `DEBUG = False`, `SECRET_KEY` & secrets από **environment**, σωστό `ALLOWED_HOSTS`.
- `gunicorn`/`uvicorn` πίσω από nginx — **ποτέ** `runserver` σε production.
- `collectstatic` + static serving, HTTPS, logging/monitoring.

## Πώς απαντάς στη συνέντευξη

«How do you build a Django REST API for production?»

> I use Django REST Framework. Serializers (`ModelSerializer`) define the exact JSON schema and handle validation, so I never leak internal fields. For CRUD I use a `ModelViewSet` plus a router, which generates all REST endpoints with correct status codes; for custom logic I drop to `APIView`/`@api_view`. Auth and permissions are declarative — `TokenAuthentication` with `IsAuthenticated` defaults, overridable per view. For production I set `DEBUG=False`, read `SECRET_KEY` and config from environment, set `ALLOWED_HOSTS`, run `collectstatic` and `migrate`, and serve with gunicorn behind nginx — never `runserver`.
