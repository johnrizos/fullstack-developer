# JavaScript Interview Drills

Αυτό το μάθημα είναι διαφορετικό: δεν είναι μόνο για να χτίσεις UI. Είναι για να απαντάς καθαρά σε συνέντευξη.

Θα δεις κλασικά patterns που ρωτάνε συχνά:

- equality και coercion,
- truthy/falsy,
- array methods,
- closure output,
- async output order.

## Drill 1: == vs ===

Το `===` συγκρίνει χωρίς type coercion. Το `==` μπορεί να μετατρέψει τύπους πριν συγκρίνει.

```js
0 == false   // true
0 === false  // false
```

Σε production κώδικα, προτίμησε σχεδόν πάντα `===`.

## Drill 2: truthy και falsy

Falsy values:

- `false`
- `0`
- `""`
- `null`
- `undefined`
- `NaN`

Όλα τα υπόλοιπα είναι truthy, ακόμα και `[]` και `{}`.

## Drill 3: array map/filter/reduce

```js
const numbers = [1, 2, 3, 4];

const result = numbers
  .filter((n) => n % 2 === 0)
  .map((n) => n * 10);
```

Πρώτα κρατάμε τα ζυγά: `[2, 4]`. Μετά τα πολλαπλασιάζουμε: `[20, 40]`.

<Exercise

  correctOptionIndex={0}
  explanation="Σωστά. Το map επιστρέφει νέο array με μετασχηματισμένα items."
/>

## Drill 4: closure output

```js
function outer() {
  let count = 0;

  return function inner() {
    count++;
    return count;
  };
}

const next = outer();
next(); // ?
next(); // ?
```

Η `inner` θυμάται το `count`, άρα οι απαντήσεις είναι `1` και `2`.

/>

## Drill 5: async output order

```js
console.log("A");

setTimeout(() => console.log("B"), 0);

Promise.resolve().then(() => console.log("C"));

console.log("D");
```

Σειρά:

1. `A`
2. `D`
3. `C`
4. `B`

Ο synchronous κώδικας τρέχει πρώτα. Τα Promise callbacks είναι microtasks και τρέχουν πριν από timer tasks όπως `setTimeout`.

<Exercise
  question="Ποια είναι η σωστή σειρά για A, setTimeout B, Promise C, D;"
  options={["A, B, C, D", "A, D, C, B", "C, A, D, B"]}
  correctOptionIndex={1}
  explanation="Σωστά. Πρώτα synchronous A/D, μετά promise microtask C, μετά setTimeout B."
/>

## Interview answer checklist

Όταν απαντάς σε JavaScript interview:

- Πες πρώτα τον κανόνα.
- Μετά δώσε μικρό παράδειγμα.
- Μετά εξήγησε το output βήμα-βήμα.
- Αν υπάρχει edge case, πες ποια είναι η ασφαλής πρακτική.

Παράδειγμα για equality:

> I prefer `===` because it avoids implicit type coercion. `==` can convert values before comparing, which can create surprising results like `0 == false` being true.
