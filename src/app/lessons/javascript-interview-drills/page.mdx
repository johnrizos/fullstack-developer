# JavaScript Interview Drills

Αυτό το μάθημα είναι διαφορετικό: είναι καθαρή προπόνηση για συνέντευξη. Κάθε drill είναι ένα πραγματικό pattern που πέφτει συνέχεια. Η μέθοδος: **πρόβλεψε → εξήγησε → επαλήθευσε**. Μην προχωράς σε επόμενο drill αν δεν μπορείς να εξηγήσεις το προηγούμενο σε 2-3 προτάσεις.

## Drill 1: == vs === και coercion

Το `===` συγκρίνει τιμή ΚΑΙ τύπο. Το `==` κάνει **type coercion** — μετατρέπει τους τύπους πριν συγκρίνει, με κανόνες που είναι γεμάτοι εκπλήξεις:

```js
0 == false        // true  — το false γίνεται 0
"" == false       // true  — και τα δύο γίνονται 0
"5" == 5          // true  — το string γίνεται number
null == undefined // true  — ειδικός κανόνας, ισούνται ΜΟΝΟ μεταξύ τους
null == 0         // false — το null ΔΕΝ γίνεται 0 εδώ!
NaN == NaN        // false — το NaN δεν ισούται με τίποτα, ούτε με τον εαυτό του

0 === false       // false — διαφορετικοί τύποι, τέλος
```

Κανόνας production: σχεδόν πάντα `===`. Η μόνη συχνή αποδεκτή χρήση του `==` είναι το `value == null`, που πιάνει `null` και `undefined` μαζί.

## Drill 2: truthy και falsy

Υπάρχουν **ακριβώς 6 falsy τιμές** (συν το BigInt `0n`) — μάθε τη λίστα απέξω:

`false`, `0`, `""`, `null`, `undefined`, `NaN`

**Όλα** τα υπόλοιπα είναι truthy — και εδώ είναι οι παγίδες:

```js
Boolean([])        // true  — empty array είναι object!
Boolean({})        // true  — empty object επίσης
Boolean("0")       // true  — non-empty string, έστω "0"
Boolean("false")   // true  — non-empty string!
Boolean(-1)        // true  — μόνο το 0 είναι falsy από τα numbers
Boolean(new Boolean(false)) // true — wrapper object, όχι primitive!
```

## Drill 3: array methods — το καθημερινό εργαλείο

Σε live coding σχεδόν πάντα θα χρειαστείς `map`, `filter`, `reduce`. Η διαφορά τους σε μία γραμμή ο καθένας:

- `map`: μετασχηματίζει **κάθε** στοιχείο → νέο array ίδιου μήκους.
- `filter`: κρατάει όσα περνούν το τεστ → νέο array μικρότερου ή ίσου μήκους.
- `reduce`: συμπυκνώνει το array σε **μία τιμή** (number, object, ό,τι θες).

```js
const orders = [
  { id: 1, total: 50, status: "paid" },
  { id: 2, total: 30, status: "pending" },
  { id: 3, total: 70, status: "paid" },
];

// Συνολικά έσοδα από πληρωμένες παραγγελίες:
const revenue = orders
  .filter((order) => order.status === "paid")
  .reduce((sum, order) => sum + order.total, 0); // 120

// Group by status — κλασικό interview task:
const byStatus = orders.reduce((groups, order) => {
  (groups[order.status] ??= []).push(order);
  return groups;
}, {});
// { paid: [...], pending: [...] }
```

Παγίδες που ρωτάνε:

```js
[1, 2, 3].map((n) => {
  n * 2;
}); // [undefined, undefined, undefined] — ξέχασες το return!

["1", "2", "3"].map(parseInt); // [1, NaN, NaN] — το map περνάει
// (value, index) και το parseInt διαβάζει το index ως radix!
```

## Drill 4: references vs copies

Τεράστιο θέμα σε interviews — τα objects και arrays περνιούνται **by reference**:

```js
const a = { count: 1 };
const b = a;          // ΔΕΝ είναι αντίγραφο — ίδιο object!
b.count = 99;
console.log(a.count); // 99

const original = [1, 2, 3];
const copy = [...original];     // shallow copy — ΟΚ για flat arrays
const deep = structuredClone(original); // deep copy — για nested δομές
```

Το spread (`...`) κάνει **shallow** copy — τα nested objects παραμένουν κοινά:

```js
const user = { name: "Ada", address: { city: "Athens" } };
const clone = { ...user };

clone.name = "Grace";          // ✅ δεν επηρεάζει το user
clone.address.city = "Patras"; // ❌ άλλαξε ΚΑΙ το user.address.city!
```

## Drill 5: closure σε loop — το αξεπέραστο κλασικό

```js
function outer() {
  let count = 0;

  return function inner() {
    count++;
    return count;
  };
}

const next = outer();
next(); // 1
next(); // 2 — το count ζει μέσα στο closure

const fresh = outer();
fresh(); // 1 — ΝΕΟ closure, νέο count
```

## Drill 6: async output order

Η συνταγή: 1) όλος ο synchronous κώδικας, 2) ΟΛΗ η microtask queue (promises), 3) ένα-ένα τα macrotasks (timers).

```js
console.log("A");

setTimeout(() => console.log("B"), 0);

Promise.resolve().then(() => console.log("C"));

console.log("D");

// A, D, C, B
```

/>

## Πώς δομείς κάθε απάντηση στη συνέντευξη

Η δομή που ξεχωρίζει τους καλούς υποψηφίους:

1. **Πες τον κανόνα** — μία πρόταση, με το σωστό όρο (microtask, TDZ, coercion, reference).
2. **Δώσε μικρό παράδειγμα** — 2-3 γραμμές κώδικα.
3. **Εξήγησε το output βήμα-βήμα** — δείξε ότι έχεις mental model, όχι αποστήθιση.
4. **Κλείσε με την ασφαλή πρακτική** — «γι' αυτό σε production χρησιμοποιώ Χ».

Παράδειγμα για equality:

> I prefer `===` because it avoids implicit type coercion. `==` converts types before comparing, which leads to surprising results — `0 == false` is true, but `null == 0` is false. The one idiomatic exception is `value == null`, which conveniently matches both null and undefined.
